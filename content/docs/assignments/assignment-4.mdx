~~~python
class BinaryHarmClassifier(FastTextClassifier):
    def __init__(
        self, model_path, positive_output, negative_output,
        positive_labels, negative_labels,
    ):
        super().__init__(model_path)
        self.positive_output = positive_output
        self.negative_output = negative_output
        self.positive_labels = {self._normalize(x) for x in positive_labels}
        self.negative_labels = {self._normalize(x) for x in negative_labels}

    @staticmethod
    def _normalize(label):
        return label.removeprefix("__label__").lower().replace("-", "_")

    def classify(self, text):
        label, score = self.predict(text)
        normalized = self._normalize(label)
        if normalized in self.positive_labels:
            return self.positive_output, score
        if normalized in self.negative_labels:
            return self.negative_output, score
        raise ValueError(f"unexpected classifier label: {label!r}")
~~~

## 1. WARC、WAT、WET 不要混

<SourceBrief label="题面译文" title="过滤 Common Crawl：题目背景">

大型语言模型主要在互联网数据上训练，但多数研究者不自行造 crawler，而使用公开抓取。最常见的是 Common Crawl：非营利组织免费提供网页语料，覆盖 17 年、超过 2500 亿页面。

把 CC dump 变成可用训练数据需要大量工作。原始网页是 HTML，需要抽文本；许多页面低质、完全或近似重复、含有害内容或敏感信息，需要删页或移除部分内容。本作业建立包含这些步骤的流水线。

实现前先观察原始数据。CC 有三种格式：

- **WARC（Web ARChive）**：原始 CC 数据，包括页面 ID/URL、metadata、HTTP request 细节（时间、服务器 IP 等）和网页原始内容。
- **WAT（Web Archive Transformation）**：从 WARC 抽出的高层 JSON metadata；HTML 可含页面 links 和 title。
- **WET（Web Extracted Text）**：从 HTML 抽取的纯文本。

题目使用 2026 Common Crawl 的 WARC/WET 样例，PDF 给出 <code>CC-MAIN-2026-12</code> 两个下载 URL；共享路径：

- <code>/shared-data/CC/example.warc.gz</code>
- <code>/shared-data/CC/example.warc.wet.gz</code>

警告：文件完全未过滤，可能有大量有害内容；遇到不愿阅读的文档可直接跳过。

**问题 look_at_cc（4 分）**

（c）好样本取决于上下文。描述一个该页面有用的应用域与一个不适合的应用域。交付 1–2 句。

WARC 的 response record 有两层：

~~~text
WARC headers
  WARC-Target-URI: https://example.org/page
  WARC-Date / WARC-IP-Address / ...

HTTP response headers

<html>原始响应字节</html>
~~~

WET 是 Common Crawl 已抽取的文本；record header 仍在，payload 已是纯文本。WET 适合大规模文本过滤，WARC 适合自行控制正文抽取，WAT 主要用于标题、链接图和 metadata。

不要手写 gzip 分隔逻辑，使用 FastWARC：

~~~python
import gzip
from fastwarc.warc import ArchiveIterator, WarcRecordType

with gzip.open("example.warc.gz", "rb") as stream:
    for record in ArchiveIterator(
        stream, record_types=WarcRecordType.response
    ):
        url = record.headers.get("WARC-Target-URI", "")
        html_bytes = record.reader.read()
        print(url, len(html_bytes))
~~~

若输入 WET，record type 改为 <code>WarcRecordType.conversion</code>。

## 2. HTML bytes → Unicode → 主文本

<SourceBrief label="题面译文" title="HTML 转主文本：完整题目">

HTML 抽文本很难。典型方法寻找可见内容，例如 <code>&lt;p&gt;</code>，但仍可能抽出浏览器可见、却非主内容的菜单和页脚。Stack Overflow 的核心是问答，但菜单、其他站点链接、注册/登录也都可见，很难可靠区分。

本作业用 Resiliparse。它还检测原始 bytes 的编码。虽然绝大多数 Web 页面是 UTF-8，流水线仍应兼容其他编码。建议用 FastWARC 的 <code>ArchiveIterator</code> 与 <code>WarcRecordType</code> 遍历。

**问题 extract_text（3 分）**

（a）函数从原始 HTML bytes 抽文本。用 <code>extract_plain_text</code>；先把 bytes 解码成 Unicode，若 UTF-8 失败则检测编码，可用 <code>detect_encoding()</code>。交付 bytes → 字符串函数，实现 <code>run_extract_text_from_html_bytes</code>，通过对应 pytest。

（b）在单个 WARC 上运行，与对应 WET 比较异同和哪种更好。交付 2–3 句。

</SourceBrief>

直接用 <code>errors="ignore"</code> 会静默丢字符。合理顺序是严格 UTF-8 快路径，失败后检测编码，仍失败才 replacement decode：

~~~python
from resiliparse.extract.html2text import extract_plain_text
from resiliparse.parse.encoding import detect_encoding

def decode_html(html_bytes: bytes) -> str:
    try:
        return html_bytes.decode("utf-8")
    except UnicodeDecodeError:
        encoding = detect_encoding(html_bytes) or "utf-8"
        return html_bytes.decode(encoding, errors="replace")

def extract_text_from_html_bytes(html_bytes: bytes) -> str | None:
    return extract_plain_text(decode_html(html_bytes))
~~~

完整代码见 [extraction.py](https://github.com/Palind-Rome/cs336-learning-journey/blob/main/examples/assignment4/cs336_data/extraction.py)。官方测试与 Moby Dick fixture 逐字比较，因此换行、空白和段落边界也是接口契约，不要在 adapter 中另做 <code>strip</code>。

## 3. 语言识别：标签、置信度与阈值

<SourceBrief label="题面译文" title="语言识别：完整题目">

Web 有数千种语言。多数 compute budget 下难以让多语言模型有效利用如此多语言，所以许多 CC 派生训练集只保留有限语言。

fastText 提供高效文本分类器、训练设施和预训练模型。<code>lid.176.bin</code> 可从 fastText 官方页下载，也位于 <code>/shared-data/classifiers/lid.176.bin</code>。语言过滤器通常依据 classifier score 保留页面；实现应返回预测置信度的非负分数。

**问题 language_identification（6 分）**

（a）接收 Unicode 字符串，返回主要语言 identifier 与 0–1 confidence。实现 <code>run_identify_language</code>，通过两个测试；测试规定英语 <code>en</code>、中文 <code>zh</code>，必要时在 adapter 重映射。

（b）训练数据过滤问题会怎样影响下游推理？高风险用户产品如何缓解？交付 2–5 句。

（c）在 WARC 抽取文本上运行，人工标 20 个随机样本并比较；报告错误、英语比例和合适阈值。交付 2–5 句。

</SourceBrief>

~~~python
import fasttext

class FastTextClassifier:
    def __init__(self, model_path):
        self.model = fasttext.load_model(str(model_path))

    def predict(self, text):
        labels, probabilities = self.model.predict(
            text.replace("\n", " "), k=1
        )
        return labels[0].removeprefix("__label__"), float(probabilities[0])

class LanguageIdentifier(FastTextClassifier):
    aliases = {"zh-cn": "zh", "zh-tw": "zh", "cmn": "zh"}

    def identify(self, text):
        label, score = self.predict(text)
        label = label.lower()
        return self.aliases.get(label, label), score
~~~

fastText 一行通常是一条样本，因此先把网页换行变空格。top-1 是英文不代表置信度足够；随机审计 20 条并对候选阈值计算：

$$
\text{precision}=\frac{TP}{TP+FP},\qquad
\text{recall}=\frac{TP}{TP+FN}.
$$

过低阈值混入其他语言，过高阈值会丢掉代码、公式、短文和混合语言。课程英语 WET 用 <code>P(en) ≥ 0.70</code> 作为起点，但它不是普适常数。高风险产品还要按语言建验证集、审计低资源语言与 code switching、版本化阈值并做下游行为测试。

<code>re.subn</code> 同时返回替换后文本与次数，正好对应 adapter：

~~~python
EMAIL_TOKEN = "|||EMAIL_ADDRESS|||"
EMAIL_PATTERN = re.compile(
    r"(?<![\w.+-])"
    r"[A-Z0-9.!#$%&'*+/=?^_\x60{|}~-]+"
    r"@[A-Z0-9](?:[A-Z0-9-]{0,61}[A-Z0-9])?"
    r"(?:\.[A-Z0-9](?:[A-Z0-9-]{0,61}[A-Z0-9])?)+"
    r"(?![\w-])",
    re.IGNORECASE,
)

PHONE_TOKEN = "|||PHONE_NUMBER|||"
PHONE_PATTERN = re.compile(
    r"(?<![\w\d])"
    r"(?:\+?1[\s.-]?)?"
    r"(?:\(\s*\d{3}\s*\)|\d{3})"
    r"[\s.-]?\d{3}[\s.-]?\d{4}"
    r"(?![\w\d])"
)

IP_TOKEN = "|||IP_ADDRESS|||"
OCTET = r"(?:25[0-5]|2[0-4]\d|1\d\d|[1-9]?\d)"
IPV4_PATTERN = re.compile(
    rf"(?<![\d.])(?:{OCTET}\.){{3}}{OCTET}(?!\d|\.\d)"
)

def mask_emails(text):
    return EMAIL_PATTERN.subn(EMAIL_TOKEN, text)

def mask_phone_numbers(text):
    return PHONE_PATTERN.subn(PHONE_TOKEN, text)

def mask_ipv4_addresses(text):
    return IPV4_PATTERN.subn(IP_TOKEN, text)
~~~

句末 <code>editor@example.com.</code> 应只替换邮箱、保留句号；<code>999.1.1.1</code> 不是 IPv4；已有 mask 不应重复计数。完整实现见 [pii.py](https://github.com/Palind-Rome/cs336-learning-journey/blob/main/examples/assignment4/cs336_data/pii.py)。

正则仍会误伤订单号/版本号，并漏掉姓名、住址、IPv6、国际号码或混淆写法。缓解方式包括上下文校验、多检测器、按域阈值、审计样本，以及高风险数据直接删除而非只掩码。

## 5. harmful content：模型输出不是道德真值

<SourceBrief label="题面译文" title="有害内容分类：完整题目">

未过滤 Web dump 含大量不希望模型复现的文本，甚至一般安全网站的用户评论也可能有害。虽然 harmful 很难划出清晰边界，数据流水线仍常过滤主要由有害内容组成的页面。

题面要求删除：少于 50 或多于 100,000 words；平均词长不在 3–10；超过 30% 非空行以省略号结尾；少于 80% words 含字母。

~~~python
import regex

WORD_PATTERN = regex.compile(
    r"[\p{L}\p{N}]+(?:['’][\p{L}\p{N}]+)*"
)

def words(text):
    return WORD_PATTERN.findall(text)

def passes_gopher_quality_filter(text):
    tokens = words(text)
    if not 50 <= len(tokens) <= 100_000:
        return False

    mean_length = sum(map(len, tokens)) / len(tokens)
    if not 3 <= mean_length <= 10:
        return False

    lines = [line.rstrip() for line in text.splitlines() if line.strip()]
    if lines:
        ellipsis_fraction = (
            sum(line.endswith("...") for line in lines) / len(lines)
        )
        if ellipsis_fraction > 0.30:
            return False

    alphabetic_fraction = (
        sum(any(ch.isalpha() for ch in token) for token in tokens)
        / len(tokens)
    )
    return alphabetic_fraction >= 0.80
~~~

边界要按文字理解：平均长度恰好 3/10 通过，恰好 30% ellipsis 通过。Unicode-aware regex 会把 <code>sentence.Next</code> 分成两个词。规则透明便宜，但会误删诗歌、代码、列表，也可能放过流畅 SEO 垃圾，所以只是 learned classifier 前的粗筛。

## 8. 精确行去重必须全局两遍

<SourceBrief label="题面译文" title="精确行去重：完整题目">

Web 有大量重复：完全相同页面、404 模板，以及 Stack Overflow 等页面间重复的 header/menu/footer。只保留全语料唯一行可消除许多模板重复，并可能留下问答等主内容。

</SourceBrief>

~~~python
def line_key(line):
    return hashlib.blake2b(
        line.encode("utf-8"), digest_size=16
    ).digest()

def exact_line_deduplicate(input_files, output_directory):
    paths = [Path(path) for path in input_files]
    output_directory = Path(output_directory)
    output_directory.mkdir(parents=True, exist_ok=True)
    frequency = Counter()

    for path in paths:
        with xopen(path, "rt", encoding="utf-8") as file:
            for line in file:
                frequency[line_key(line)] += 1

    for path in paths:
        with (
            xopen(path, "rt", encoding="utf-8") as source,
            xopen(
                output_directory / path.name,
                "wt", encoding="utf-8",
            ) as target,
        ):
            for line in source:
                if frequency[line_key(line)] == 1:
                    target.write(line)
~~~

128-bit hash 碰撞概率极低但非零；更严格系统可在同 hash bucket 内再核对原文，超大语料可用外部排序或分片 counter。

## 9. MinHash + LSH：只对候选做真实比较

<SourceBrief label="题面译文" title="MinHash 与 LSH 文档去重：完整题目">

精确方法无法处理只改年份、作者或空白的模板文档，因此模糊文档级去重使用 MinHash 与 LSH。

文档表示为 n-gram 集合，集合 $S,T$ 的 Jaccard 是 $|S\cap T|/|S\cup T|$。全量两两计算是二次复杂度，存所有 n-gram 集合也耗内存。

MinHash 使用 $k$ 个不同 hash 函数。对集合 $S=\{s_1,\ldots,s_n\}$，第 $i$ 维为 $\min(h_i(s_1),\ldots,h_i(s_n))$。两个集合的 Jaccard 可由签名相同维度比例近似；例如 [1,2,3,2] 与 [5,2,3,4] 有 2/4 维相同。

LSH 避免仍比较全部签名对。把 $k$ 维分为 $b$ bands、每 band $r$ 行，$k=br$。任一 band 全同就进同 bucket，成为 candidate duplicate。固定签名长度时，更多 bands 提高 recall、降低 precision。题面的两个六维签名用 3×2 切分，第一 band [1,2] 相同，因此成为候选，其他 bands 不匹配也不取消候选。

候选可再算真实 Jaccard，超过阈值才判重复。重复关系还要跨 bucket 聚类：A-B 与 B-C 通过时，即使 A-C 未直接配对，也组成 A/B/C 簇，最终每簇保留一篇。

**问题 minhash_deduplication（8 分）**

函数至少输入：文件路径列表、MinHash 数、LSH bands 数、word n-gram 长度、Jaccard threshold 和输出目录。可假设 hash 数能被 band 数整除。为提高 recall，按 RefinedWeb 做 lowercase、去标点、统一空白、去 accents、Unicode NFD。

输出只写非候选/未达阈值文档，以及每个重复簇选中的一篇，并保留原文件名。实现 <code>run_minhash_deduplication</code>，通过测试。

</SourceBrief>

规范化 word n-gram 集合后定义：

$$J(S,T)=\frac{|S\cap T|}{|S\cup T|}.$$

~~~python
def normalize_for_deduplication(text):
    text = unicodedata.normalize("NFD", text.lower())
    text = "".join(
        ch for ch in text
        if unicodedata.category(ch) != "Mn"
    )
    text = re.sub(r"[^\w\s]", " ", text)
    return " ".join(text.split())
~~~

先 NFD 再去 combining marks，<code>é</code> 才变为 <code>e</code>。对随机 hash $h$：

$$
\Pr[\min h(S)=\min h(T)]
=\frac{|S\cap T|}{|S\cup T|}
=J(S,T).
$$

用 $k$ 个 hash，签名相同维度比例估计 Jaccard。再把 $k=br$ 维切为 $b$ bands、每 band $r$ 行。相似度为 $s$ 的两文档成为候选的概率：

$$P(\text{candidate}\mid s)=1-(1-s^r)^b.$$

固定 $k$，增加 bands 会提高 recall、降低 precision。实现：

~~~python
def candidate_pairs(signatures, num_bands):
    rows = len(signatures[0]) // num_bands
    buckets = defaultdict(list)
    candidates = set()
    for doc_index, signature in enumerate(signatures):
        for band in range(num_bands):
            start = band * rows
            key = (
                band,
                signature[start:start + rows].tobytes(),
            )
            for previous in buckets[key]:
                candidates.add((previous, doc_index))
            buckets[key].append(doc_index)
    return candidates
~~~

LSH collision 只是候选信号，之后必须算真实规范化 n-gram Jaccard。A≈B、B≈C 时用 union-find 合成同一簇。课程允许随机保留一个；示例为可复现性按排序路径保留每簇第一个。完整实现见 [deduplication.py](https://github.com/Palind-Rome/cs336-learning-journey/blob/main/examples/assignment4/cs336_data/deduplication.py)。

### 10.3 统计怎样解释

按顺序短路时：

$$
\text{drop fraction}_i =
\frac{\text{first removed by filter }i}{\text{seen}}.
$$

这是 first-hit 计数。同一文档若同时 low-quality 和 toxic，只归到先执行者。研究过滤器重叠时要另跑 shadow evaluation：全部打标签但暂不删除。

<code>filter-stats.json</code> 保存 wall seconds、worker seconds、input bytes 和 bytes/s。全 dump 粗估：

$$
T_{\text{full}}\approx
\frac{\text{full dump bytes}}{\text{measured bytes per second}}.
$$

报告机器、worker 数、存储介质、是否排除下载时间，以及小样本外推假设；不能把冷启动吞吐伪装成稳态吞吐。

## 12. tokenize：流式写 uint16，文档间加 EOS

<SourceBrief label="题面译文" title="Tokenize：完整题目">

使用 GPT-2 tokenizer 把过滤数据编码为整数 ID，每篇后加入 <code>&lt;|endoftext|&gt;</code>。starter 用 multiprocessing 编码、flatten，转 <code>np.uint16</code> 后通过 <code>ids_array.tofile(output_path)</code> 写文件。

**问题 tokenize_data（2 分）**

编写 tokenize/序列化脚本；必须使用与 starter 兼容的 uint16 <code>tofile</code> 格式。报告过滤数据总 token 数。

</SourceBrief>

GPT-2 vocabulary 小于 65,536，token ID 可保存为 <code>np.uint16</code>。每篇末尾必须加 <code>&lt;|endoftext|&gt;</code>，否则网页 A 尾部与网页 B 开头会被误当成同一文档。

课程 starter 把所有 token 放进列表，大数据会耗尽内存。可运行实现按文档立即写：

~~~python
ENCODING = None

def initialize_worker():
    global ENCODING
    ENCODING = tiktoken.get_encoding("gpt2")

def encode_document(text):
    if ENCODING is None:
        initialize_worker()
    return ENCODING.encode_ordinary(text) + [ENCODING.eot_token]

def tokenize_jsonl(input_paths, output_path, workers=None, chunksize=100):
    token_count = 0
    with (
        Path(output_path).open("wb") as output,
        multiprocessing.Pool(
            workers or multiprocessing.cpu_count(),
            initializer=initialize_worker,
        ) as pool,
    ):
        for token_ids in pool.imap(
            encode_document,
            iter_jsonl_text(input_paths),
            chunksize=chunksize,
        ):
            array = np.asarray(token_ids, dtype=np.uint16)
            array.tofile(output)
            token_count += len(array)
    return token_count
~~~

~~~bash
python -m cs336_data.cli tokenize \
  --input "data/deduplicated/*.jsonl" \
  --output data/train.bin \
  --workers 32
~~~

完成后检查：

~~~python
ids = np.fromfile("data/train.bin", dtype=np.uint16)
assert len(ids) == reported_token_count
assert ids.max() < 50257
print(tokenizer.decode(ids[:2000]))
~~~

EOS 数应大致等于文档数。若解码出现大量 JSON 的 URL/key，说明错误地编码了整个 object，而不是 <code>record["text"]</code>。
