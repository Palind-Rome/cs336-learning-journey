## 每一章怎样带你完成作业

每道核心题都沿同一顺序展开：

1. **先读题面译文**：明确题目给了什么、要求返回什么、有哪些限制和实验问题。
2. **建立最小心智模型**：用小 shape 或一条数据流解释算法为什么存在。
3. **写出完整实现**：页面给出可运行代码，并标出每个变量对应的公式、shape 与状态。
4. **跟一次执行轨迹**：用玩具输入逐步检查索引、mask、归约、通信或参数更新。
5. **运行窄测试再扩展**：先验证一个接口，再接入训练循环、性能实验或完整流水线。

五份实现也分别位于 `examples/assignment1` 到 `examples/assignment5`。阅读代码时，不要只停在“能运行”：先预测下一行产生的 shape 和不变量，再实际执行；完成一节后关闭页面，独立复写关键函数并用相同测试核对。

## 通用循环

1. 阅读 PDF 对应小题，写下输入、输出和不变量。
2. 打开 `tests/adapters.py`，确认测试真正调用的签名。
3. 预测一个最小样例的结果或 shape。
4. 对照本章完整实现，逐段运行并解释每个中间变量，只跑一个窄测试。
5. 失败时先看第一个不同值、最大绝对误差、dtype/device 和广播轴。
6. 正确后再做性能、并行或大规模实验。

```bash
# 在对应 assignment 目录内
uv run pytest tests/test_model.py -k rmsnorm -v
```
