主作业使用 <code>allenai/OLMo-2-0425-1B</code> 和 GSM8K，目标是让模型通过可验证 reward 学会数学 reasoning：

~~~text

  → REINFORCE
  → group baseline / std advantage / sequence normalization
  → on-policy GRPO
  → Dr.GRPO / RFT / MaxRL
~~~

~~~text
  → frozen reference + DPO
  → capability / helpfulness / safety trade-off
~~~

## 0. 任务、模型、数据与符号

主作业模型 OLMo-2-0425-1B 预训练/中期训练共见约 4T tokens；下游 GSM8K 是小学数学文字题。选择这个组合的原因是：1B 模型训练成本可控，但长数据训练使它已经能偶尔探索出正确 reasoning。RL 的学习动力来自“偶尔采到正 reward”；若初始模型从不答对，所有 rollout 都是 0，group advantage 也全是 0。

## 1. Prompt baseline：先知道模型原本会什么

### 1.1 三种 prompt 改变的是行为分布

- <code>question_only</code>：只给题目，并要求最终答案放在 `\boxed{}`；
- <code>r1_zero</code>：明确要求 <code>&lt;think&gt;...&lt;/think&gt;</code> 与 <code>&lt;answer&gt;...&lt;/answer&gt;</code>；
- <code>r1_zero_three_shot</code>：在同一结构前放三组完整示例。

zero-shot 并不代表“没有 prompt”，而是没有示范样本；CoT instruction 仍然很长。few-shot 既展示任务，也展示格式和 reasoning 风格，可能提高可解析率，但会占 context、诱导照抄形式。

vLLM generation 参数按题面固定：

~~~python
sampling_params = {
    "temperature": 1.0,
    "top_p": 1.0,
    "max_tokens": 512,
}

# 只给 r1_zero 系列；question_only 不使用
sampling_params["stop"] = ["</answer>"]
sampling_params["include_stop_str_in_output"] = True
~~~

温度 1 与 top-p 1 表示从完整模型分布采样，这对 RL exploration 很重要；baseline accuracy 因此本身有 sampling variance。

### 1.2 grader 契约

GSM8K 的 ground truth 是 `{rationale} #### {answer}`，reward function 需要纯 answer：

~~~python
def extract_gsm8k_answer(answer_with_rationale):
    return answer_with_rationale.rsplit("####", 1)[-1].strip()
~~~

<code>r1_zero_reward_fn</code> 解析 answer tags；<code>question_only_reward_fn</code> 找 boxed answer。两者返回 total、format、answer reward。本实验不因格式正确给部分总分：

~~~text
total reward = answer reward
format reward = only a logging / diagnosis metric
~~~

基线报告至少分三类：

1. format=1、answer=1；
2. format=1、answer=0；
3. format=0、answer=0。

每类还要人工看 parser error。category 2 可能数学正确但等价表达解析失败；category 3 可能回答正确但完全未遵循格式。不能把 parser accuracy 当模型 reasoning accuracy。

完整评测 helper 在 [evaluation.py](https://github.com/Palind-Rome/cs336-learning-journey/blob/main/examples/assignment5/cs336_alignment/evaluation.py)：保留 prompt、response、三个 reward、吞吐和平均长度，JSONL 可复查。

## 2. 从 expected reward 推导 REINFORCE

目标是模型自己的采样正确率：

$$
J_\theta
=\mathbb E_{x\sim\rho}
  \mathbb E_{y\sim\pi_\theta(\cdot\mid x)}
  [r(y\mid x)].
$$

难点是 sampling distribution 也依赖 $\theta$。用 log-derivative trick：

$$
\nabla_\theta \pi_\theta(y\mid x)
=\pi_\theta(y\mid x)\nabla_\theta\log\pi_\theta(y\mid x),
$$

得到：

$$
\nabla_\theta J_\theta
=\mathbb E_{x,y}
\left[
r(y\mid x)\nabla_\theta\log\pi_\theta(y\mid x)
\right].
$$

而 autoregressive response：

$$
\log\pi_\theta(y\mid x)
=\sum_{t=1}^{L}
\log\pi_\theta(y_t\mid x,y_{<t}).
$$

所以一条正确 response 的同一个 scalar reward 会乘到它所有 response tokens；错误 response 在原始 REINFORCE 中权重 0。

给 $B$ 道题、每题 $G$ 个 rollout 的 Monte Carlo estimator：

$$
\hat g
=\frac1{BG}\sum_{i=1}^{B}\sum_{j=1}^{G}
r(y^{(i,j)}\mid x^{(i)})
\nabla_\theta\log\pi_\theta(y^{(i,j)}\mid x^{(i)}).
$$

## 3. Baseline 到底为什么能减方差

减去不依赖当前 sampled response/action 的 baseline $b$：

$$
\mathbb E[(r-b)\nabla\log\pi]
=\mathbb E[r\nabla\log\pi]
-b\,\mathbb E[\nabla\log\pi].
$$

而 score function 的期望为：

$$
\mathbb E_{y\sim\pi}[\nabla\log\pi(y)]
=\nabla\sum_y\pi(y)=\nabla1=0.
$$

所以期望不变，但 variance 可能升也可能降。

### 3.1 题面 Bernoulli 例题完整解

令 $A\in\{0,1\}$，$\Pr(A=1)=p=\sigma(\theta)$，$r(A)=A$。Bernoulli log-policy score：

$$
\nabla_\theta\log\pi_\theta(A)=A-p.
$$

无 baseline 的单样本随机变量 $Z=r(A)(A-p)$：

| A | 概率 | Z |
|---:|---:|---:|
| 1 | $p$ | $1-p$ |
| 0 | $1-p$ | 0 |

因此：

$$
\mathbb E Z=p(1-p),\qquad
\operatorname{Var}(Z)=p(1-p)^3.
$$

$n$ 个 iid 样本平均的方差：

$$
\boxed{\operatorname{Var}(\hat g)=\frac{p(1-p)^3}{n}}.
$$

加入 baseline 后 $Z_b=(r-b)(A-p)$：

| A | $Z_b$ |
|---:|---:|
| 1 | $(1-b)(1-p)$ |
| 0 | $bp$ |

化简得：

$$
\boxed{
\operatorname{Var}(\hat g_b)
=\frac{p(1-p)(b-(1-p))^2}{n}
}.
$$

这个玩具问题的最优常数 baseline 是 $b^*=1-p$，不是 reward population mean $p$。若题目指定 $b=p$：

$$
\operatorname{Var}(\hat g_{b=p})
=\frac{p(1-p)(2p-1)^2}{n}.
$$

它在 $0<p<2/3$ 时低于无 baseline，$p>2/3$ 时反而更高，在边界相等。这正说明“减均值一定降低方差”不是普适定理。

### 3.2 GRPO 的 group mean 有有限样本缩放

同题 $G$ 个 response 的均值 $\mu=\frac1G\sum_jr_j$ 依赖这一组 sampled responses，所以不满足普通 action-independent baseline 条件。把自项与其他项展开后：

$$
\mathbb E\left[
\frac1G\sum_j(r_j-\mu)\nabla\log\pi(y_j)
\right]
=\frac{G-1}{G}\nabla J.
$$

方向保持但乘 $(G-1)/G$；$G$ 越大越接近原梯度。组内相对 reward 就叫 advantage。

## 4. GRPO 的三个归一化选择

标准 GRPO：

$$
A^{(i,j)}
=\frac{r^{(i,j)}-\mu_i}{\operatorname{std}_i+\varepsilon}.
$$

课程实现要求 <code>torch.std</code> 默认 sample std（分母 $G-1$），不是 population std。reward [1,0] 的 mean=.5、sample std≈.7071，所以 advantage≈±.7071。

<AdvantageNormalizerLab />

std division 不再保持原 objective 的期望；它使各组更新尺度更接近，但会重加权不同难度 prompt。全对或全错组减均值后 advantage 全 0，没有学习信号——这也是 group sampling 必须兼具探索的原因。

标准 GRPO 还对每条 sequence 先除自身长度：

$$
\frac1{BG}\sum_{i,j}
\frac1{L_{ij}}
\sum_t A^{(i,j)}
\nabla\log\pi_\theta(y_t^{(i,j)}\mid x_i,y_{<t}).
$$

这样每条 response 总权重接近相等，但长 response 的每 token 权重更小。Dr.GRPO 会移除 std normalization，并把全 batch token loss 除固定常数。

## 5. 张量契约：先把 off-by-one 全消掉

### 5.1 prompt/output 分开 tokenize

不加 BOS、EOS 或 separator，直接拼接。对 combined token 序列 $z$：

~~~text
input_ids = z[:-1]
labels    = z[1:]
~~~

<code>response_mask</code> 与 labels 对齐，不与 input_ids 对齐。若 prompt 长 $P$，第一个 response token 是 combined[$P$]，它出现在 labels[$P-1$]，所以 mask 从 $P-1$ 开始。

~~~python
def tokenize_prompt_and_output(prompts, outputs, tokenizer):
    pairs = []
    for prompt, output in zip(prompts, outputs, strict=True):
        prompt_ids = tokenizer.encode(prompt, add_special_tokens=False)
        output_ids = tokenizer.encode(output, add_special_tokens=False)
        pairs.append((prompt_ids + output_ids, len(prompt_ids)))

    max_len = max(len(ids) for ids, _ in pairs)
    padded = torch.full(
        (len(pairs), max_len),
        tokenizer.pad_token_id,
        dtype=torch.long,
    )
    mask = torch.zeros((len(pairs), max_len - 1), dtype=torch.bool)
    for row, (ids, prompt_len) in enumerate(pairs):
        padded[row, :len(ids)] = torch.tensor(ids)
        mask[row, prompt_len - 1 : len(ids) - 1] = True
    return {
        "input_ids": padded[:, :-1],
        "labels": padded[:, 1:],
        "response_mask": mask,
    }
~~~

### 5.2 per-token log probability 与 entropy

~~~python
def get_response_log_probs(model, input_ids, labels, return_token_entropy=False):
    logits = model(input_ids).logits
    log_dist = F.log_softmax(logits, dim=-1)
    selected = log_dist.gather(-1, labels.unsqueeze(-1)).squeeze(-1)
    result = {"log_probs": selected}
    if return_token_entropy:
        probabilities = log_dist.exp()
        result["token_entropy"] = -(probabilities * log_dist).sum(-1)
    return result
~~~

selected log-prob 是真实 sampled token 的 $\log p$；entropy 是整个 next-token distribution 的 $-\sum_vp_v\log p_v$，两者不能混淆。函数返回所有 positions，mask 在聚合时排除 prompt/padding。

### 5.3 reward 与 group normalization

~~~python
def compute_group_normalized_rewards(
    raw_rewards, group_size,
    baseline="mean",
    advantage_eps=1e-6,
    advantage_normalizer="std",
):
    groups = raw_rewards.reshape(-1, group_size)
    means = groups.mean(1, keepdim=True)
    advantages = groups - means if baseline == "mean" else groups.clone()

    if advantage_normalizer == "std":
        advantages /= groups.std(1, keepdim=True) + advantage_eps
    elif advantage_normalizer == "mean":
        advantages /= means + advantage_eps
    elif advantage_normalizer != "none":
        raise NotImplementedError
    return advantages.reshape_as(raw_rewards)
~~~

完整、已测实现见 [grpo.py](https://github.com/Palind-Rome/cs336-learning-journey/blob/main/examples/assignment5/cs336_alignment/grpo.py)。

## 6. Policy-gradient loss：负号、ratio 与 mask

PyTorch optimizer 做 gradient descent，所以 loss 是 policy objective 的负数。

### 6.1 On-policy

每条 sequence 的 scalar advantage 广播到 response tokens：

~~~python
per_token_loss = -advantages[:, None] * policy_log_probs
~~~

不要在这里乘 response mask；保持 loss shape 为 [B,S]，由唯一的 aggregation 函数统一 mask，能避免两处 mask 逻辑不一致。

### 6.2 两种 aggregation

标准 sequence normalization：

~~~python
masked = per_token_loss * response_mask
per_sequence = masked.sum(-1) / response_mask.sum(-1)
loss = per_sequence.mean()
~~~

constant normalization：

~~~python
loss = masked.sum() / normalization_constant
~~~

constant 模式不能先在 microbatch 内平均再除常数，否则 batch 切法会改变梯度。

### 6.3 Gradient accumulation 精确缩放

sequence 模式中，microbatch 已对自身 sequences 取 mean。要还原 full batch mean，第 $m$ 个 microbatch loss 乘：

$$\frac{\text{microbatch sequences}}{\text{original batch sequences}}.$$

constant 模式每个 microbatch 已是“该部分 token sum / 同一个全局常数”，直接逐个 backward 后相加，不再乘 microbatch fraction。

~~~python
optimizer.zero_grad(set_to_none=True)
for microbatch in microbatches:
    micro_loss = ...
    if loss_normalization == "sequence":
        backward_loss = micro_loss * len(microbatch) / full_batch_size
    else:
        backward_loss = micro_loss
    backward_loss.backward()

gradient_norm = torch.nn.utils.clip_grad_norm_(model.parameters(), 1.0)
optimizer.step()
optimizer.zero_grad(set_to_none=True)
~~~

最后 <code>set_to_none=True</code> 使所有 <code>parameter.grad is None</code>，也避免下一 batch 意外累积旧梯度。课程 tests 不只比较 scalar loss，还 snapshot 一步后的所有参数，因此负号、sample std、microbatch 权重和固定分母任何一个错都会被发现。

### 6.4 zero-advantage pruning

advantage 恰好为 0 的 sequence 对梯度贡献严格为 0，可不做 model forward：

- group-mean baseline：一组全对或全错时整组为 0；
- RFT 无 baseline：所有 reward=0 的 response 为 0。

但分母不能跟着改。sequence 模式仍按 original batch size 加权；constant 模式仍除原 fixed $Z$。示例代码先在完整组上计算 advantage，再 prune，保留数学等价。

## 7. 完整 on-policy GRPO loop

Algorithm：

1. 从 GSM8K 采 $B=32$ prompts；
2. 每题采 $G=8$，共 256 responses；
3. grader 得 256 个 rewards；
4. 同题组内算 advantages；
5. 256 responses 分 32 个 microbatches 累积梯度；
6. clip、AdamW step、清梯度；
7. 把更新权重同步到 vLLM；
8. 周期性在至少 1024 validation examples 上评测。

推荐参数：

~~~python
n_train_examples = 6400
n_val_examples = 1024
num_rollout_steps = 200
learning_rate = 1e-5
rollout_batch_size = train_batch_size = 256
group_size = 8
gradient_accumulation_steps = 32
sampling_temperature = 1.0
sampling_max_tokens = 512
max_grad_norm = 1.0
optimizer = torch.optim.AdamW(
    policy.parameters(),
    lr=learning_rate,
    betas=(0.9, 0.95),
    weight_decay=0.0,
)
~~~

[training.py](https://github.com/Palind-Rome/cs336-learning-journey/blob/main/examples/assignment5/cs336_alignment/training.py) 的 <code>run_grpo_updates</code> 接受课程 vLLM server，执行权重同步、rollout、old log-prob、on/off-policy minibatches 和经过官方 snapshot 的 train step。

训练日志至少包括：

| 指标 | 看什么 |
|---|---|
| train/val total reward | 真正 accuracy 是否上升 |
| train/val format reward | 提升是否只是更会输出 tags |
| loss | policy surrogate，不保证像 CE 一样单调下降 |
| gradient norm | 爆炸、长期为 0、clip 是否常触发 |
| token entropy | exploration 是否过早坍缩 |
| response length | 是否靠冗长/截断投机 |
| pass@k | 一题 $k$ 次至少一次正确的探索能力 |
| sampled rollouts | reward hacking、格式漂移、reasoning 质量 |

先跑约 50 steps sanity check。reward 开始接近 0 很正常；但长期所有组全 0，模型就没有梯度，需要检查 prompt、grader、采样和初始 exploration。

正式比较用 4 seeds。每条 seed 轨迹都应保留；可画 mean 与原始轨迹，或 mean ± $1.96\,s/\sqrt n$。只有 4 seeds 时 normal interval 很粗，不能把小差异描述成确定结论。

## 8. Dr.GRPO、RFT、MaxRL 逐项比较

| 方法 | baseline | advantage normalizer | loss normalizer |
|---|---|---|---|
| GRPO | group mean | group sample std | sequence |
| GRPO_constant | group mean | group sample std | fixed constant |
| Dr.GRPO | group mean | none | fixed constant |
| RFT | none | none | fixed constant |
| MaxRL | group mean | group mean | fixed constant |

### 8.1 Sequence length normalization 的取舍

按 sequence length 除：每条 response 总权重相近，减少长回答主导 batch；但长 reasoning 每个正确 token 被系统性降权，会偏向短 answer，并改变原 expected-reward gradient。

固定常数：每个 response token 权重一致，更接近原 policy gradient；但长 response 总贡献更大，长度分布或截断会显著影响 gradient norm。固定 $Z=BG L_{\max}$ 也让有效 token 少的 batch 更新更小，可能更稳定但学习率含义随 padding/长度改变。

### 8.2 RFT 与 Dr.GRPO

binary reward 下，RFT objective：

$$
\frac1Z\sum_{x,j}\mathbf1[r_j=1]\log\pi_\theta(y_j\mid x).
$$

其 on-policy 期望与 REINFORCE 相同。Dr.GRPO 减 sampled group mean，有限 $G$ 时是原梯度的 $(G-1)/G$ 缩放，通常因 baseline 方差更低。

行为差别：

- 一组全正确：Dr.GRPO advantage 全 0；RFT 继续强化全部正确样本；
- 一组有对有错：Dr.GRPO 提高正确并显式降低错误；RFT 只给正确正权重；
- 一组全错：两者都无更新；
- RFT 可 prune 大量错误 response，吞吐更高，但正例极少时梯度稀疏。

### 8.3 三个 difficulty reweightings 的推导

定义当前 policy 在 prompt $x$ 上的成功率：

$$\eta(x)=\mathbb E_{y\sim\pi_\theta}[r(y\mid x)].$$

$G\to\infty$ 且 binary reward 时：

| 方法 | surrogate 中的 prompt weight $w(x)$ |
|---|---|
| Dr.GRPO | $1$ |
| GRPO ÷ std | $1/\sqrt{\eta(x)(1-\eta(x))}$ |
| MaxRL ÷ mean | $1/\eta(x)$ |

推导要点：无限组 mean 变为 $\eta$，是只依赖 $x$ 与 stop-gradient policy 的 baseline；binary reward std 是 $\sqrt{\eta(1-\eta)}$。normalizer 作为不求导的 prompt-dependent scalar 直接成为 $w(x)$。

GRPO 强调接近全错或全对的 prompts，但有限组中这些组常 advantage 全 0；MaxRL 极强地放大低成功率 prompts，需 $\varepsilon$ 防零除。Dr.GRPO 保持均匀 prompt objective，但不同 prompt 的 gradient scale 差异更大。不存在脱离模型/数据的绝对胜者。

## 9. Off-policy：为什么 stale rollout 需要 ratio

behavior policy $\pi_0$ 生成 rollout，更新后的 current policy 是 $\pi_\theta$。朴素 estimator：

$$
\mathbb E_{y\sim\pi_0}
[r(y)\nabla\log\pi_\theta(y)]
$$

通常不等于 current-policy gradient。无偏 sequence importance sampling：

$$
\mathbb E_{y\sim\pi_0}
\left[
\frac{\pi_\theta(y\mid x)}{\pi_0(y\mid x)}
r(y)\nabla\log\pi_\theta(y\mid x)
\right].
$$

但 sequence ratio 是：

$$
\frac{\pi_\theta(y\mid x)}{\pi_0(y\mid x)}
=\prod_{t=1}^{L}
\exp\left(
\log\pi_\theta(y_t\mid x,y_{<t})
-\log\pi_0(y_t\mid x,y_{<t})
\right),
$$

长度一大就可能极小或极大，variance 爆炸。所有实现都应从 log-ratio 开始，不能先把小 probabilities 相除。

### 9.1 Token-level PPO/GRPO ratio

用每 token：

$$w_t=\exp(\log\pi_\theta(y_t)-\log\pi_0(y_t)).$$

unclipped loss：

~~~python
ratio = torch.exp(policy_log_probs - old_log_probs)
loss = -(advantages[:, None] * ratio)
~~~

它不等于无偏 sequence ratio。其 surrogate 可理解为：第 $t$ 个 token 从 current policy 采，prefix 与 suffix 仍来自 old policy。策略漂移越远，这个偏差越严重，但 variance 远低于完整乘积。

若每两 token 一组做 pairwise ratio，对应 surrogate 是对每个 pair $(2t-1,2t)$，这两个位置来自 $\pi_\theta$，其余位置来自 $\pi_0$ 的 expected rewards 之和。一般 $k$-token block 同理。

### 9.2 PPO clipping 的符号不能写反

$$
\min(Aw_t,\ A\,\operatorname{clip}(w_t,1-\epsilon,1+\epsilon)).
$$

<ImportanceClipLab />

当 $A>0$：ratio 超过 $1+\epsilon$ 后 clipped 分支为常数，gradient 归零；当 $A<0$：ratio 低于 $1-\epsilon$ 后归零。不是简单把 ratio clamp 后始终反传；<code>min</code> 与 advantage 符号共同决定哪一侧停止。

~~~python
unclipped = advantage * ratio
clipped = advantage * ratio.clamp(1 - epsilon, 1 + epsilon)
surrogate = torch.minimum(unclipped, clipped)
loss = -surrogate
clip_fraction = (unclipped != surrogate).float().mean()
~~~

CISPO 的可选方案直接 upper-bound gradient importance weight，超上界后仍有非零 gradient；它控制 variance 的方式与 PPO trust-region 风格目标不同。

### 9.3 GSPO 几何平均

GSPO 给整条 response 一个共享 ratio：

$$
s
=\exp\left[
\frac1L\sum_{t\in response}
(\log\pi_\theta(y_t)-\log\pi_0(y_t))
\right].
$$

这就是 sequence ratio 的 $1/L$ 次方，减少长度指数方差，但引入 bias。只在 response mask 上取平均，prompt 与 padding 不能进入。

~~~python
log_ratio = policy_log_probs - old_log_probs
counts = response_mask.sum(1, keepdim=True)
sequence_ratio = torch.exp(
    (log_ratio * response_mask).sum(1, keepdim=True) / counts
)
unclipped = advantage * sequence_ratio
clipped = advantage * sequence_ratio.clamp(1 - epsilon, 1 + epsilon)
loss = -torch.minimum(unclipped, clipped).expand_as(policy_log_probs)
~~~

展开后由 sequence aggregation 取平均，gradient 自然包含 $1/L$。课程默认 token GRPO cliprange=.2，GSPO cliprange=3e-4；数值不可直接比较，因为 ratio 定义不同。

### 9.4 Bias–variance 光谱

| 方法 | bias | variance | 直觉 |
|---|---|---|---|
| no reweighting | 高，策略漂移后更高 | 最低 | update 很小或模型几乎未变时可能够用 |
| token ratio + clip | 中 | 中低 | 局部修正，每 token 可独立 clip |
| GSPO geometric sequence + clip | 仍有 bias | 中 | 共享 sequence fate，和 response-level reward 更一致 |
| full sequence product | 理论无偏 | 极高 | 长 LLM response 通常不可用 |

32× off-policy 配置把 rollout 256 拆为 train batches 8、grad accumulation 1；同一 inference batch 连做 32 updates。要在生成时保存 $\pi_0$ 的 old log-probs，并保持 tokenization、response 顺序、padding 列完全对齐。日志新增 clip fraction，观察后段 minibatches 是否几乎全被 clip。

## 10. 自己提出 estimator 时怎样做有效实验

只改一个因素，并找到一个已有 baseline 与它仅差这一项。例如：

> 用 difficulty-capped MaxRL：normalizer 为 $\max(\mu,\tau)$，其他完全等于 constant-normalized MaxRL。

直觉是保留对困难 prompt 的加权，同时限制 $\mu\to0$ 的极端放大。预先写清：

1. 公式与 stop-gradient 部分；
2. 对应已有 baseline；
3. 预期改善的 failure mode；
4. 新 hyperparameter $\tau$ 的选择规则；
5. 同 prompts、seeds、预算和 logging；
6. 不只报最佳 seed，给全部轨迹与 uncertainty。

结果没超过 baseline 也有价值，只要实现正确、假设清晰、分析能说明哪里不符合预期。

## 11. Safety 补充：从 zero-shot 到 SFT

| benchmark | 测量 | metric |
|---|---|---|
| MMLU | 多学科事实/推理 | option accuracy、parse failures |

### 11.1 严格 parser

MMLU 只接受指定句式中的 A–D：

~~~python
pattern = re.compile(

)
~~~

GSM8K 取输出最后一个 numeric token，支持负号、小数与逗号：

~~~python
matches = re.findall(
    r"[-+]?(?:\d{1,3}(?:,\d{3})+|\d+)(?:\.\d+)?",
    output,
)
prediction = matches[-1].replace(",", "") if matches else None
~~~

不要把 option 文本里偶然出现的字母或 reasoning 中间数当最终 prediction。实现见 [metrics.py](https://github.com/Palind-Rome/cs336-learning-journey/blob/main/examples/assignment5/cs336_alignment/metrics.py)。

### 11.2 Packed SFT 数据

每条 prompt/response 套 Alpaca template，文档末尾加 Llama EOS。全部文档 token 串接后，按 seq length $m$ 的连续窗口：

~~~text
stream: 0 1 2 3 4 5 6 7 8 ...
item 0 input : 0 1 2 3
item 0 label : 1 2 3 4
item 1 input : 4 5 6 7
item 1 label : 5 6 7 8
~~~

最后不足 $m+1$ 的部分丢弃。两个 items 在边界共享一个 token，是因为前一 item 的最后 label 同时是后一 item 的第一个 input；没有丢 next-token transition。

~~~python
self.num_examples = (len(token_ids) - 1) // seq_length

def __getitem__(self, index):
    start = index * self.seq_length
    return {
        "input_ids": tokens[start:start + seq_length],
        "labels": tokens[start + 1:start + seq_length + 1],
    }
~~~

[data.py](https://github.com/Palind-Rome/cs336-learning-journey/blob/main/examples/assignment5/cs336_alignment/data.py) 与课程 Llama tokenizer fixture 的 75 个 packed examples 逐 token 一致。

SFT 训练：一 epoch、context 512、effective batch 32、lr 2e-5、cosine decay、3% linear warmup、weight decay .1、grad clip 1。gradient accumulation 的最后不足 $k$ 个 microbatches 要除实际 group length，而不是仍除 $k$；[training.py](https://github.com/Palind-Rome/cs336-learning-journey/blob/main/examples/assignment5/cs336_alignment/training.py) 已处理这一点。

## 12. HH preference 与 DPO

HH 四个文件含 chosen/rejected conversations。题面要求忽略 human 发过多次消息的 multi-turn，因为两条 conversation 后续 human text 可能已分叉。loader：

1. chosen/rejected 都必须恰有一个 <code>Human:</code>；
2. 分出相同 instruction；
3. 分出两个 assistant responses；
4. 保存 source split，供 helpful/harmless 分析。

### 12.1 从 RLHF 到 DPO objective

传统 RLHF 先拟合 reward model：

$$
\ell_r=-\log\sigma(r_\phi(x,y_w)-r_\phi(x,y_l)),
$$

再用 PPO 优化 policy，并常加 reference KL 与预训练辅助 loss。DPO 利用 KL-regularized RL 的最优 policy 关系：

$$
r(x,y)
=\beta\log\frac{\pi_r(y\mid x)}{\pi_{\rm ref}(y\mid x)}
+\beta\log Z(x).
$$

chosen/rejected 做差时 $Z(x)$ 消失：

$$
\ell_{\rm DPO}
=-\log\sigma\left[
\beta\left(
\log\frac{\pi_\theta(y_w\mid x)}{\pi_{\rm ref}(y_w\mid x)}
-\log\frac{\pi_\theta(y_l\mid x)}{\pi_{\rm ref}(y_l\mid x)}
\right)
\right].
$$

$\beta$ 越大，log-ratio margin 在 sigmoid 中被放大；从原 KL-regularized 解释看，它也对应离 reference 的惩罚尺度。reference 必须 frozen。

### 12.2 response log-prob 与 prompt cancellation

chosen/rejected 使用同一 prompt。对同一 model：

$$
\log\pi(x,y_w)-\log\pi(x,y_l)
=\log\pi(y_w\mid x)-\log\pi(y_l\mid x),
$$

因为 $\log\pi(x)$ 精确抵消。因此可以把完整 Alpaca-formatted prompt+response（末尾 EOS）分别求 sequence log-prob 再相减。必须在 policy device 返回 loss；reference scalar 从另一 GPU 搬到 policy device，reference forward 用 no-grad。

~~~python
policy_margin = logp_policy_chosen - logp_policy_rejected
with torch.no_grad():
    reference_margin = logp_ref_chosen - logp_ref_rejected
logit = beta * (policy_margin - reference_margin.to(policy_margin.device))
loss = -F.logsigmoid(logit)
~~~

完整实现见 [dpo.py](https://github.com/Palind-Rome/cs336-learning-journey/blob/main/examples/assignment5/cs336_alignment/dpo.py)，官方 tiny GPT-2 fixture 得 loss 0.9104。

DPO 推荐 2 GPUs：policy/reference 各一份 SFT checkpoint；留约 200 validation；effective batch 64、$\beta=.1$、RMSprop lr 1e-6、训练一 epoch，保存 validation preference accuracy 最高 checkpoint。这里的 classification accuracy 是 policy 是否给 chosen 更高 log-prob，不等于 AlpacaEval 或 safety。

最后同时评估 AlpacaEval、SimpleSafetyTests、MMLU、GSM8K。前两者直接对应 HH helpful/harmless，后两者检测 alignment tax。任何单一 judge 或单一 benchmark 都不能证明模型“已安全”。

### 第 2–3 页：为什么 post-training 需要 RL

前四次作业构建 base model；现在要把它变成能解决下游任务的工具。alignment/instruction tuning 把拥有广泛知识和行为的 base model 聚焦为 helpful、harmless chat model；optional supplement 讲这些技术。

另一个 post-training 组成是 RL。预训练追求广覆盖，使用广泛 Web 文本；post-training RL 聚焦特定任务的高准确率。差异至少有：可用数据更少，目标从 coverage 变为 precision，因此需要新技术。

RL 给 questions 与 scoring function，却不给要模仿的正确 responses。代码题可能用 tests 判答案；数学题解析 final answer。训练从模型采样 responses、评分、提高正确 response 的权重，直接对 accuracy 取梯度。

RL 慢且不稳定；不同 seeds 方差高，小实现细节影响大，所以本作业同时学习数学与实证挑战。

模型 OLMo-2-0425-1B 在 OLMo-mix-1124 预训练、Dolmino-mix-1124 mid-train，总计 4T tokens。前者以 DCLM-Baseline 为主；后者约一半 DCLM，一半 instruction/math/code/STEM/wiki。更多见 OLMo 2 report。

### 第 4–6 页：Prompting 与 grading

Prompting 是把 base model 推向 task-solving behavior 的轻量方法，也会塑造 RL exploration。

vLLM 提供高效 generation，包括 CUDA kernels 与 PagedAttention KV cache。starter <code>VLLMServer</code> 有 model id、GPU、seed、memory utilization；<code>start()</code> 启动；<code>generate_completions(prompts,sampling_params,batch_size)</code> 返回 text、token_ids、finish_reason。

默认 <code>r1_zero</code> prompt 描述 User/Assistant conversation，要求 reasoning 放 <code>&lt;think&gt;</code>，answer 放 <code>&lt;answer&gt;</code>；模板已包含 opening think tag，模型应关闭 think、输出 answer 并关闭 answer。tags 便于 parse 与遇到 <code>&lt;/answer&gt;</code> 时 stop。

### 第 6–9 页：GRPO 目标、LM policy 与 REINFORCE

目标：

$$J_\theta=\mathbb E_{x\sim\rho}\mathbb E_{y\sim\pi_\theta(y\mid x)}[r(y\mid x)].$$

它不同于预训练 cross entropy，因为内层样本来自模型本身。RL 高层流程是 sample、grade、reinforce；CoT prompt 与 RL 的结合可显著提升强 base model reasoning。

传统 RL 的 state $s_t$、action $a_t$、reward $r_t$、next state 对应 LM prefix、next token、终点 grader 与 prefix 拼接。LM 是 categorical stochastic policy，必须支持 sampling 与对 sampled action 计算 log likelihood。

trajectory $\tau=(s_0,a_0,\ldots,s_T,a_T)$；prompt 从 $\rho$ 采样，token 逐步拼接，直到 EOS、answer close tag 或 max budget。trajectory/episode/rollout 同义。

verified math domain 中间 reward 为 0，terminal action 得 1（grader 判 final 正确）或 0。return 就是 terminal reward。优化 $\theta^*=\arg\max_\theta J_\theta$。

gradient ascent 为 $\theta_{k+1}=\theta_k+\alpha\nabla J$。log-derivative trick 推导：

$$
\nabla J
=\mathbb E[r(y\mid x)\nabla\log\pi_\theta(y\mid x)].
$$

直觉：从数据采题、从模型采回答，平均 reward-weighted log-prob gradient，提高高 reward response 概率。推导先把 expectation 写为对 y 求和，用 $\nabla\pi=\pi\nabla\log\pi$，再写回 expectation；prompt expectation 与 gradient 可交换。

$B$ prompts、每题 $G$ iid responses 的 estimator 是 $1/(BG)$ 的双重平均。

### 第 9–12 页：Baselines、advantage、std 与 sequence normalization

REINFORCE 无偏但高方差。减常数 $b$ 后正确 response 若 $r=1$ 得 $1-b$，错误 response 得 $-b$；只要 $b$ 不依赖当前 response，利用 expected score=0 可证明期望不变，但 variance 可能增减。

**问题 baseline_calcs（5 分）**

（b）推导 baseline $b$ estimator 的 variance，交付 $n,b,p$ 表达式与讨论。

（c）代入 population mean $b=p$，与无 baseline 比较：总是低、总是高还是依 p 而异？

GRPO 第一项是 group mean baseline $\mu_i=G^{-1}\sum_jr_{ij}$。虽然依赖 sampled y，展开可证明期望是原 gradient 的 $(G-1)/G$。这些减均值 reward 称 advantage。

之后除 group std。题面公式展示 population-style 定义，但明确实现用 <code>torch.std</code> 默认 sample std，并加 epsilon。除 std 不再保留原 expected-reward objective；可视为稳定技巧，使 iid Gaussian 假设下各组 update norm 接近。

再把 sequence log-prob sum 除自己的 response length。这样长 sequence 的 token 相对短 sequence 被下权，进一步改变 expectation。标准 GRPO 按原论文保留它。

importance reweighting 稍后处理；on-policy 一 inference batch 只一步，不需 ratio。

Algorithm 1：初始化 policy；每 step 采 $B$ questions、每题 $G$ outputs；计算 mean/std 与 length-normalized policy gradient；用 optimizer 更新 $\theta$。注意 optimizer 实现 loss 时要取负号。

### 第 31–32 页：Off-policy 与 importance sampling

此前每个 inference batch 一次 update，完全 on-policy。off-policy 对一批 rollout 做多 train steps，可能更快，但不稳定且复杂。

把 256 responses 分 minibatches 连续 step 后，第一 step 之后 current $\pi_\theta$ 不再等于 inference $\pi_0$，样本 stale。直接把 stale y 当 current samples 的 estimator 期望错误。

importance weight $\pi_\theta(y\mid x)/\pi_0(y\mid x)$ 可恢复 current-policy expectation：current 下更可能的 stale sample 上权，低概率者下权。但 weight 上界 $C$ 可使 variance 放大 $C$；LM sequence weight 是 $L$ 个 token ratios 的乘积，随 length 指数恶化。因此无权 estimator 高 bias/无额外 ratio variance，完整 reweighting 零 bias/高 variance，后续方法在两者间折中。

### 第 32–35 页：Token-level ratio 与 PPO clipping

PPO/GRPO 用 token-level ratio 代替 sequence product：

$$
\mathbb E_{y\sim\pi_0}
\left[r(y)\sum_t
\frac{\pi_\theta(y_t\mid x,y_{<t})}{\pi_0(y_t\mid x,y_{<t})}
\nabla\log\pi_\theta(y_t\mid x,y_{<t})\right].
$$

它不等于 principled sequence estimator，但 ratio 不随 L 乘积，variance 大幅降低。

定义 surrogate policy $\tilde\pi_t$：除位置 t 来自 current $\pi_\theta$，其他所有位置来自 old $\pi_0$。固定 response length L 时，token estimator 优化 $\sum_tE_{y\sim\tilde\pi_t}r(y)$。因此 prefix 并非 current policy 分布，current token 对未来的影响也由 old-policy suffix 评估；$\pi_\theta$ 离 $\pi_0$ 越远，bias 越严重。sequence ratio 会修正 prefix/suffix，但 variance 高。

**问题 derive_surrogate_objectives（2 分）**

给出 pairwise importance estimator：每项把 $(2t-1,2t)$ 两个 token 的 ratios 相乘，并对两位置联合 log probability 求梯度，其他 tokens 来自 old policy。推导它优化的 surrogate objective，交付表达式与过程。

为限制 current 离 old 太远，最简单是少做 updates；clipping 试图按 batch 自适应挤出更多 steps。题面实现 PPO 主流 clipping。

先将 token ratio 加到 standard GRPO objective；再令 $A^{ij}$ 是 sequence advantage、$w_t$ token ratio：

$$
J_{\rm clip}
=\frac1{BG}\sum_{ij}\frac1{L_{ij}}\sum_t
\min(A^{ij}w_t,\ A^{ij}\operatorname{clip}(w_t,[1-\epsilon,1+\epsilon])).
$$

若 $A\ge0$，目标用 $\min(w,1+\epsilon)A$；若 $A<0$，用 $\max(w,1-\epsilon)A$。正 advantage 被提高到 ratio $1+\epsilon$ 后 gradient=0；负 advantage 被压到 $1-\epsilon$ 后 gradient=0。gradient mask 因 advantage 符号不同而方向不同。

**问题 compute_policy_gradient_loss_off_policy（1 分）**

PPO clipping 也可解释为压制极端 ratio variance。CISPO 更直接只对 gradient weight 上限 clip 为 $1+\epsilon$，超过上限仍继续有 gradient；可选与其他方法比较。

### 第 35–37 页：GSPO

token-level reweighting忽略 prefix/suffix。GSPO 论文认为 GRPO 不稳定，改用 sequence-level ratio，但把完整 product 取 $1/L$ 次方，即几何平均：

$$
s^{ij}
=\left(\prod_t
\frac{\pi_\theta(y_t\mid x,y_{<t})}
     {\pi_0(y_t\mid x,y_{<t})}
\right)^{1/L_{ij}}.
$$

objective 对整条 sequence 使用共享 $s$ 并做 PPO-style min/clip。去掉 $1/L$ 与 clipping 就回到 principled sequence ratio；几何平均和 clip 用 bias 换低 variance。对 $s$ 求 gradient 自然产生 $1/L\sum_t\nabla\log\pi_\theta(y_t)$，因此已经有 sequence length normalization。若把 GSPO 用到 fixed-constant RL variant，需要相应处理 exponent。

**问题 think_about_importance_reweighting（2 分）**

### 第 3 页：MMLU 零样本基线

MMLU 是多项选择题，但自由文本生成可能返回选项字母、完整选项文字，甚至改写后的答案，直接评分并不简单。因此提示词必须规定固定输出格式。任务提示词如下：

~~~text
回答下面关于 {subject} 的多项选择题。只用一句“The correct answer is _”回答，
把空白处替换成正确答案对应的字母（A、B、C 或 D）。

Question: {question}
A. {options[0]}
B. {options[1]}
C. {options[2]}
D. {options[3]}
Answer:
~~~

其中 **subject** 是 MMLU 子领域，例如高中地理；**question** 是题干；**options** 是四个选项。先用样本填充任务提示词，再把完整任务提示词放进系统提示词的 **instruction** 占位符。评测时从生成文本中解析预测字母并和标准答案比较。生成必须使用贪心解码：temperature 0.0、top-p 1.0。

### 第 8 页：观察 SFT 数据与 Alpaca 模板

**问题 look_at_sft：检查指令微调数据（4 分）**

实现 SFT 时，先用 **prompts_safety/alpaca_sft.prompt** 把提示—回答对转为字符串。它与前面的零样本系统提示不同：

~~~text
Below is an instruction that describes a task. Write a response that appropriately completes the request.

### Instruction:
{instruction}

### Response:
{response}
~~~

把每条格式化字符串当作一个语言建模文档，在文档之间加入分隔符（例如 Llama 3.1 8B 的 end-of-text token），把所有文档连接成一个 token 序列。数据加载器从中生成批次：每批有 B 个长度 m 的输入序列，以及对应的长度 m 的下一 token 标签。

实践中会把样本打包成定长序列，以减少 padding、提高 GPU 吞吐。对 token 流切连续、不重叠的长度 m 块；最后不足 m 的块丢弃。例如 token IDs 为 [0,1,2,…,9,10]，长度为 4 时，可能的输入依次是 [0,1,2,3] 与 [4,5,6,7]。完整遍历加载器时，每块恰好返回一次，构成一个 epoch。

### 第 10 页：SFT 训练脚本与梯度累积

用 Hugging Face Transformers 加载模型；用 bfloat16 与 FlashAttention-2 节省显存：

~~~python
from transformers import AutoModelForCausalLM, AutoTokenizer

tokenizer = AutoTokenizer.from_pretrained(model_name_or_path)
model = AutoModelForCausalLM.from_pretrained(
    model_name_or_path,
    torch_dtype=torch.bfloat16,
    attn_implementation="flash_attention_2",
)
~~~

把 input_ids 与 labels 移至设备，前向取 **model(input_ids).logits**，再用交叉熵计算语言建模损失。训练结束时对模型与分词器都调用 **save_pretrained(output_dir)**，使目录可以独立加载。

即使采用 bfloat16 和 FlashAttention-2，单卡也未必装得下目标有效批量。上下文 512 通常只能用较小的设备内批量，而题面希望每次参数更新相当于 32 条序列。梯度累积把大批量分成多个 microbatch：若分成 k 次，先把每次 loss 除以 k 再 backward，只在每 k 次之后调用 optimizer.step 与 zero_grad。这样累积的是平均梯度，有效批量扩大 k 倍。

### 第 14 页：RLHF 的完整流程

SFT 让模型模仿高质量示范，但不一定消除预训练阶段学到的不良行为。对齐时，可以让待改进模型自己针对提示生成回答，再按质量和适当性奖励或惩罚。

RLHF 即从人类反馈中进行强化学习。先准备一组提示，让 SFT 模型对每条提示生成一组回答。“RL”来自训练信号不再是每个 token 的参考答案损失，而是衡量整段回答质量的标量奖励；“HF”表示原始方法通过拟合人类标注数据获得奖励：标注者人工排序候选回答。

原始 RLHF 很复杂。SFT 后，为每个提示生成 K 个回答并让人排序，这在规模化时很昂贵。随后显式拟合奖励模型 rθ(x,y)：它给提示 x 下的回答 y 一个标量奖励。模型通常从移除最终输出层的 SFT 模型开始，再接标量输出层。对偏好数据中的优胜回答 yw 与落选回答 yl，优化：

$$
\ell^r_\theta(x,y_w,y_l)=-\log\sigma\big(r_\theta(x,y_w)-r_\theta(x,y_l)\big).
$$

奖励排序和人工排序越一致，损失越小。拟合奖励模型后，把语言模型视作策略 πθ：它接收提示，每步选择一个 token，回答结束时从 rθ 获得奖励。原论文用 PPO 优化策略；还加入相对 SFT 参考模型的 KL 惩罚以限制漂移，并加入辅助预训练语言建模目标，避免下游性能退化。

RLHF 组件很多，据报道很难复现 OpenAI 的成功。DPO 因简单且有效而流行，常能达到或超过 RLHF。最后一部分将实现 DPO，并用偏好标签对齐模型。

### 第 15 页：DPO 目标

RLHF 先学奖励模型，再寻找高奖励策略。DPO 观察到：不用先求最优奖励 r、再求该奖励下的最优策略 πr，可以把最优奖励重参数化为最优策略本身：

$$
r(x,y)=\beta\log\frac{\pi_r(y\mid x)}{\pi_{\mathrm{ref}}(y\mid x)}+\beta\log Z(x).
$$

πref 是不希望训练策略偏离太远的原 SFT 参考策略；β 控制偏离参考策略的惩罚强度；Z(x) 只依赖指令 x，不依赖回答 y。奖励模型损失只依赖两个回答的奖励差，因此 Z(x) 抵消，得到单样本 DPO 损失：

$$
\ell_{\mathrm{DPO}}=-\log\sigma\left[
\beta\log\frac{\pi_\theta(y_w\mid x)}{\pi_{\mathrm{ref}}(y_w\mid x)}-
\beta\log\frac{\pi_\theta(y_l\mid x)}{\pi_{\mathrm{ref}}(y_l\mid x)}
\right].
$$

对齐期间无需像 RLHF 那样在线采样，只需计算条件对数概率，所以没有显式强化学习。偏好也不一定由人产生；已有研究使用其他语言模型判断同一问题的成对回答。

Anthropic 的 HH（Helpful and Harmless）数据包含四个训练集合：**harmless-base、helpful-online、helpful-base、helpful-rejection-sampled**。文件分别是：

~~~text
data/hh/harmless-base.jsonl.gz
data/hh/helpful-base.jsonl.gz
data/hh/helpful-online.jsonl.gz
data/hh/helpful-rejection-sampled.jsonl.gz
~~~

每个 gzip 文件都是 JSON Lines；每项包含人类偏好的 chosen 对话和 rejected 对话，两者从相同提示开始。

### 第 16 页：HH 处理与 DPO 损失实现

**问题 look_at_hh：检查 HH 偏好数据（2 分）**

1. 编写函数合并四个文件。忽略人类发送超过一条消息的多轮对话，因为它们后续的人类内容可能已分叉；把每项拆成 instruction、chosen assistant response、rejected assistant response；保留来源文件名用于分析。交付 HH 加载函数，gzip 与 json 模块会有帮助。
2. 研究者没有硬性定义“helpful”或“harmless”，而让标注者自己解释。各随机查看三条 helpful 与 harmless，说明 chosen/rejected 的主要差别以及是否赞同标签；交付二至四句话。

实现 DPO 单样本损失时，输入包括待优化语言模型、参考模型，以及同一提示 x 下的 yw、yl。两个大模型可能位于不同设备，返回的 loss 必须位于待优化模型的设备。

同一模型下比较 log π(yw|x)−log π(yl|x) 时，提示 x 的概率会抵消，所以它等价于连接字符串 concat(x,yw) 与 concat(x,yl) 的无条件对数概率之差。

**问题 dpo_loss：DPO 损失（2 分）**

实现单样本 DPO loss。用 **alpaca_sft.prompt** 格式化提示和回答，并在每个回答后追加 EOS token。在适配器实现 **run_compute_per_instance_dpo_loss**，运行 **uv run pytest -k test_per_instance_dpo_loss**。交付损失函数。

### 第 17 页：DPO 训练与 alignment tax

DPO 需要把 chosen/rejected 分别送入 πref 与 πθ，显存开销明显，因此题面采用不做样本批处理、只做梯度累积的简单实现。除非用量化等额外技巧，否则不用 AdamW，而按原始 DPO 工作使用 RMSprop。

建议路线：

1. 使用两张 GPU，参考模型与训练模型各占一张；
2. 在两张卡上各加载一份指令微调后的模型；
3. 留出约 200 条作为验证集；
4. 用 DPO loss 与梯度累积训练，并跟踪每步损失；
5. 起始超参数为有效批量 64、β=0.1、学习率 1e-6。

还要记录隐式奖励模型在验证集上的“分类准确率”：若 chosen completion 的对数概率高于 rejected completion，就把该项记为分类正确。

**问题 dpo_training：DPO 训练（1 B200 小时，4 分）**

1. 实现 DPO 训练循环，在 HH 上训练 SFT Llama 一个 epoch；保存验证准确率最高的检查点。交付训练脚本与验证准确率截图。
2. 按 alpaca_eval_sft 评估 DPO 模型，报告普通/长度控制胜率，并与 SFT 模型比较。
3. 在 SimpleSafetyTests 上评估并与 SFT 模型比较。
4. AlpacaEval 与 SimpleSafetyTests 都直接测 HH 中展示过的行为，例如遵循指令和拒绝潜在有害提示。既有工作常观察到 **alignment tax（对齐税）**：对齐后某些能力可能下降。再评估 GSM8K 与 MMLU，用二至三句话报告观察。
