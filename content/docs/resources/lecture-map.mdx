## Lecture 4 的架构岔路

标准 softmax attention 需要 `T×T` 交互。线性 attention 尝试用结合律 `Q(KᵀV)` 或递推状态降低长序列成本；Mamba/DeltaNet 引入可选择更新的状态；混合架构保留少量 full attention 补足精确检索。

MoE 则沿另一维扩展：每个 token 只路由到 Top-K 专家，使总参数量大而每 token FLOPs 较小。代价是离散路由训练、负载均衡、all-to-all 通信和容量管理。dense Transformer 是理解这些方案的基线；先问“省了什么资源，又把复杂度移到哪里”。

## Lecture 11 的实战补丁

μP（Maximal Update Parametrization）试图让不同宽度模型的激活与更新保持合适尺度，从小模型迁移学习率等超参；WSD（warmup-stable-decay）让多个实验共享稳定训练段，再分叉不同 decay，节约缩放实验。但 norm gain、weight decay、优化器与架构变化会破坏理想迁移，必须用 pilot 验证。
