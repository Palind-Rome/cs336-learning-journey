CS336 的全名是 **Language Modeling from Scratch**。它不像"调用一个大模型 API"的课程，而像操作系统课：把一个复杂系统拆开，再亲手装回去。课程的主线只有一句话：

> 在有限的数据、算力、显存和时间下，怎样造出尽可能好的语言模型？

你会依次经历文本编码、Transformer、优化训练、GPU kernel、分布式、缩放律、数据工程、评测和后训练。五次作业恰好对应这条生产链，而不是五座互不相干的小岛。

<KnowledgeBridge insight="你用过 AI 聊天、翻译、写代码。但你想过没有——这一切实质的起点，是同一个极其简单的问题：'下一个字应该是什么？'">

这门课不教你怎么**用**模型，而是教你怎么**造**模型。不用先成为专家。你只需要好奇心和愿意反复问"为什么"。每一章都以一个日常直觉开始，再带你进入公式和代码——因为最好的学习路径，不是先背术语，而是先知道**要解决什么问题**。

## 你需要什么基础？

不用先成为 LLM 专家。你最好见过 Python、矩阵乘法、导数和基本概率；不熟也可以从「零号站」补齐。真正重要的是愿意反复回答三件事：**张量形状是什么？资源花在哪里？测试证明了什么？**

## 怎么使用本站

1. **先读直觉**：能用自己的话解释问题，再看公式。
2. **手算形状**：任何神经网络 bug，先检查每个轴代表什么。
3. **只实现一个接口**：运行最小测试，确认后再向外扩。
4. **写实验日志**：记录假设、改动、指标和结论，不凭印象调参。

作业章节按"题面译文 → 必要先修 → 完整实现 → 逐段解释 → 测试与排错"组织。先读清楚函数为什么存在，再运行参考实现并追踪一次最小输入，最后关闭文档独立复写关键路径。
