## Lecture 15：模型从“续写”变成“助手”

Pretraining 学会“什么文本可能接在后面”；assistant 需要的是“在这个角色、规则与工具集合下，什么响应最合适”。Instruction following 是一种极强的控制接口：同一模型可按自然语言切换任务，但这种控制不是凭空出现，而是由示范、偏好与优化规则共同塑造。

### 1. SFT 的真正目标是 behavior extraction

开放 SFT 数据从 FLAN 的任务模板/短答案，走向 Self-Instruct/Alpaca 的合成指令、ShareGPT/Vicuna 的长对话、OpenAssistant 的人工细节、WizardLM/Tülu 的混合，再到 Nemotron 一类带 tool call、todo 与 agent trace 的数据。变化不只是“数量更多”：response length、bullet style、引用、知识密度、工具 schema、safety 与多轮状态都在变。

SFT 的标准 loss 是 response-only conditional likelihood。对 chat template 生成的完整序列 $z=(x,y)$：

$$
\mathcal L_{\mathrm{SFT}}
=-\sum_{t\in\mathcal A}\log\pi_\theta(z_t\mid z_{<t}),
$$

其中 $\mathcal A$ 只包含 assistant 内容和必要的 assistant control/end tokens。System/user/tool-output tokens 作为 context 输入，但 labels 置为 ignore；多轮对话可只训最后一轮，也可训所有 assistant spans，必须显式声明。

~~~python
def response_only_labels(input_ids, assistant_mask, ignore_index=-100):
    labels = input_ids.clone()
    labels[~assistant_mask] = ignore_index
    # 模型 logits[:, t] 预测 input_ids[:, t+1]；框架内部通常完成 shift。
    return labels
~~~

常见 silent bug 是把 user prompt 也计 loss，使模型学习复述用户；把 padding/end mask 错一位；packing 多个 conversations 时允许 attention 穿越样本；chosen/rejected 后续使用不同 chat template。先在一个 batch 打印 role、token、label 三列，确认每个 assistant span 才开始大训练。

SFT 最擅长把 pretraining 已有行为“提取”成可调用格式：听从指令、简洁/详细风格、结构化 JSON、工具调用、安全拒绝。它不一定适合把 tail facts 硬写进参数。若 target response 包含模型原本不知道的事实/引用，teacher forcing 只奖励逐 token 模仿；部署时模型可能学到“被问引用就生成一个长得像引用的字符串”，却没学会何时知道它是真的。研究和课程经验都提示：在 unknown facts 上 fine-tune 可能增加 hallucination；可验证 correctness reward、retrieval 或继续 pretraining 更合适。

### 2. Style、事实与 safety 要分开做消融

偏好 judge——无论人类还是 GPT——对长度有强倾向，因而长 response 的 SFT data 会显著抬高 Arena/AlpacaEval，却未必提高 MMLU 等任务。Length、bullet points、礼貌开场和 citation-like style 是容易学习的 surface features；事实正确性、推理与 calibration 更难。应同时报告 length-controlled preference、task accuracy、citation verification 与 response-length histogram。

Safety SFT 往往用真实用户场景提取 taxonomy，再为 harmful/non-harmful、borderline、benign transformation 等情况写合规响应。公开细节稀少，Llama 2 量级只披露约几千例；Tülu 3 公布的 pipeline 混合 CoCoNot、WildJailbreak、WildGuardMix 等安全/不合规数据。讲义展示的研究中，约 500 条恰当样本就能显著改善 hate-speech、Anthropic HH 等安全指标，向 Alpaca 加 500 条也能教出 guideline following。

少量样本改变大行为，是因为它在抽取已有能力；但 safety long tail 仍需更多覆盖。只训拒绝会 over-refusal，只训明显恶意会漏掉变体，只按关键词会误伤医学/历史讨论。数据表至少标 harm category、intent、allowed transformation、locale、response policy 与 grader evidence。

### 3. Mid-training：把小而尖的 SFT 变成可扩展 mixture

只有 SFT 时，海量 instruction data 用高 LR 多 epoch 可能 catastrophic forgetting；而 pretraining 只见 web continuation，又离 chat format 太远。Two-phase/mid-training recipe 是：

1. 在 web/pretraining data 上训练；
2. 在后段把 instruction、domain、long-context 或 code data 混入 next-token training；
3. 最后做短而高质量的 response-masked SFT。

Mid-training 与 pretraining 可以用同一个 causal LM objective，区别在 mixture、serialization、schedule 与 checkpoint 目标。Instruction pair 可保留 user/assistant markers，并选择对整段或仅 assistant 计 loss；前者更像 pretraining、吞吐高，后者更贴近 SFT。MiniCPM、JetMoE 等公开了类似 two-phase recipe，更多公司将其视作常识但披露有限。

### 4. 为什么从 imitation 转向 preference

SFT 拟合一个示范分布 $p^*(y\mid x)$，必须有人/teacher 写出答案。Preference optimization 则寻找使可测 reward 最大的 policy。Generation–verification gap 说明 annotator 可能不会写一篇最佳摘要，却能在两篇中挑较好者；同样，unit tests 能验证代码而无需先写参考 patch。

标准 pairwise record 是 $(x,y_w,y_l)$：同一 prompt 下 chosen 与 rejected。Rubric 不能只写“更好”，而要规定 correctness、relevance、helpfulness、style、safety 的优先级，允许 tie/两者皆差，并记录 confidence/rationale。随机交换左右位置、隐藏 model identity、对事实题提供检索/工具、放入 gold checks 与重复题，可测 annotator reliability。

Crowdsourcing 的难处是验证 worker 是否真正核对 correctness、是否偷偷使用 AI、补偿与任务压力是否导致草率，且大规模标注有劳务/心理健康伦理。Annotator demographics 会系统性改变价值取向；即使聚合很多人，expert 与 crowd 对 contradiction、inconsistency、factuality、relevance、format/repetition 的 error profile 也不同。最终 reward 是“这批 annotators 在此 guideline 下的偏好”，不是抽象全人类偏好。

AI feedback 可扩展：GPT-4 在系统级 win-rate 上曾与人类近乎 rank-perfect correlation，逐例 agreement 约接近 human inter-annotator；UltraFeedback、Zephyr、Tülu/OLMo 都使用 LM preference。Constitutional AI 让模型先按原则 critique/revise，再比较并训练 harmlessness preference/RLAIF。优势是规模和一致 rubric；风险是 self-preference、position/length/style bias，以及把 teacher 的盲点固化进 student。

### 5. 显式 reward model 与 PPO 的三层约束

Bradley–Terry reward model 令：

$$
P(y_w\succ y_l\mid x)=\sigma(r_\phi(x,y_w)-r_\phi(x,y_l)),
$$

并最小化负对数似然。Reward 只有差值可辨识；整体加常数不变。训练后对 policy rollout 打分，RLHF objective 通常写：

$$
\max_{\pi}\ \mathbb E_{x,y\sim\pi}[r_\phi(x,y)]
-\beta D_{\mathrm{KL}}(\pi(\cdot\mid x)\|\pi_{\mathrm{ref}}(\cdot\mid x)),
$$

有时再混入 pretraining/SFT loss 防能力回退。KL 不是装饰：reward model 在标注数据附近拟合，policy 若远离 reference 会进入 reward extrapolation region，学会 exploitation。

Policy gradient 用 $\nabla\mathbb E[R]=\mathbb E[R\nabla\log\pi]$，方差高；TRPO 在线性化 surrogate 下加 trust-region KL constraint；PPO 用 old policy ratio $\rho_t=\pi_\theta(a_t|s_t)/\pi_{old}(a_t|s_t)$ 并 clip：

$$
L^{clip}=\mathbb E_t\left[
\min(\rho_tA_t,\operatorname{clip}(\rho_t,1-\epsilon,1+\epsilon)A_t)
\right].
$$

LM PPO 还需 reward/value heads、token-level KL、advantage/return、rollout workers、reference/old policy 与多轮 minibatch，因而“公式看起来无害，系统很难调”。Lecture 16 会继续展开。

### 6. DPO：把 reward 写成 policy log-ratio

对上面的 KL-regularized objective，若 $\pi$ 可取任意分布（关键的 nonparametric assumption），每个 prompt 的闭式最优解为：

$$
\pi^*(y\mid x)=\frac1{Z(x)}\pi_{ref}(y\mid x)e^{r(x,y)/\beta}.
$$

反解：

$$
r(x,y)=\beta\log\frac{\pi^*(y\mid x)}{\pi_{ref}(y\mid x)}+\beta\log Z(x).
$$

把它代入 Bradley–Terry chosen/rejected reward 差，$\log Z(x)$ 抵消。用参数 policy 代替 $\pi^*$，得到：

$$
\mathcal L_{DPO}=-\mathbb E\log\sigma\!\left(
\beta\left[
\log\frac{\pi_\theta(y_w|x)}{\pi_{ref}(y_w|x)}-
\log\frac{\pi_\theta(y_l|x)}{\pi_{ref}(y_l|x)}
\right]\right).
$$

DPO 省掉显式 RM 与 on-policy rollout/outer loop，但没有省掉假设：pairwise data 必须代表目标 policy 附近的比较，reference 固定，sequence logprob 定义一致。它本质是对隐式 pairwise reward 做 MLE，并通过另一种参数化间接优化 policy。

求和 logprob 天生让长度进入 margin；SimPO 删除 explicit reference 并加 margin，length-normalized DPO 除以各 response token 数。它们改变统计模型，不只是数值稳定技巧。公开模型也常把 DPO 与 expert iteration 组合：当前模型采样→reward/filter→偏好更新→再采样。

### 7. Reward 越高之后发生什么

PPO 并非一定输给 DPO；Tülu 3 等实验显示结果高度依赖 data、RM、β/KL、sampling 与 compute。要防两个共同失败：

- **Overoptimization**：proxy reward 先随优化上升，真实 human preference 或独立 judge 达峰后下降。课程图中 human preference 与 noisy LM preference 都出现，noiseless synthetic preference 没出现，说明 reward error 是根因之一。
- **Mode collapse/calibration loss**：policy 把概率集中到少数高奖励格式，长度和模板化上升，entropy 降低。RLHF 后模型不再自动是对自然文本校准的 probabilistic model；MMLU confidence reliability curve 与长度分布都会变化。

训练仪表应同时画 train RM、held-out human/judge、KL、entropy、response length、n-gram/self-BLEU diversity、base capability 与 safety slices。最佳 checkpoint 往往不是 reward 最大的最后一步。

**第 32 页。** 从 imitation 到 optimization。SFT 拟合 $\hat p(y|x)\approx p^*(y|x)$，是纯 generative-modeling 视角且需要 reference-policy samples。RLHF 寻找最大化 $E_p[R(y,x)]$ 的 policy；它优化可测 reward，LM 此时被视作 policy，而不是某自然分布的模型。

**第 33 页。** 为什么优化？Generation–verification gap：人不总能写出自己更偏好的 LM output。新闻摘要研究中，不同 annotator 对 reference 与 Instruct-Davinci summary 的偏好不同，总体 agreement 只有0.07，说明写 reference 与选择偏好不是同一件事。

**第 34 页。** RLHF 分三方面：data——怎样收集与担忧什么；算法——PPO、DPO；side effects——优化后的副作用。

**第 35 页。** RLHF data 问什么样的 pairwise feedback，以及怎样获得好的 pairwise feedback；流程图聚焦 InstructGPT 的 comparison→reward-model 阶段。

**第 36 页。** 标准 pairwise setup 给 annotator 同一 instruction/input 下两个 responses，按 guidelines 选择 preferred，并可能给 ranking/理由。截图展示 response 1/2 与 rating 选项。

**第 37 页。** InstructGPT guideline 要标注者按 usefulness、truthfulness、harmlessness 等标准排序，面对明显错误、含糊、拒答、风格与事实核查时遵循详细规则；页面展示多页原始说明。

**第 38 页。** 另一个旧例是 Bard annotation guideline，分别规定 instruction-following、正确性、相关性、写作质量与 safety，要求比较 responses 并给理由；讲义附 Bloomberg 文档链接。

**第 41 页。** Crowdsourcing 的复杂性：难找到真正高质量且可验证 annotators；难让其认真检查 correctness；还要防未声明 AI use。

**第 42 页。** 大规模 data collection 有显著伦理问题。页面引用肯尼亚 workers 低薪处理 toxic ChatGPT 文本，以及美国已经存在“AI underclass”的报道，涉及低薪、心理伤害与不透明劳动。

**第 43 页。** RLHF annotator demographics 会显著改变行为。Santurkar 2023 表格按 political、religion、race、gender 等群组比较模型与不同人群 agreement，显示聚合偏好不是中立常数。

**第 44 页。** Annotators 对 style/错误类型影响很大。Hosking、Blunsom、Bartolo 2024 对 crowd/expert 比较 contradiction、inconsistency、factuality、relevance、formatting、repetition；experts 往往更能发现 inconsistency/factuality，某些类型差异较小，因此“expert 总在所有维度更好”也不成立。

**第 45 页。** GPT-4 是出乎意料好的 pairwise feedback system：system-level simulated win-rate 与 human win-rate Spearman correlation 0.98、$R^2=0.87$；instance-level agreement 接近 human inter-annotator 水平。

**第 48 页。** Length effect 是 RLHF 的显著结果。Chen 2024 图显示多种方法随输出 length 增加 reward/ranking；Singhal 2024 例子表明 judge 对长细节答案与短直接答案的偏好会显著影响训练。

**第 49 页。** 已经有高质量 pairwise pipeline，怎样让模型使用？第一部分是原始且难调的 PPO 简版；第二部分是较新且易用的 DPO。

**第 50 页。** 再次对比 imitation 与 optimization：SFT 拟合 reference distribution，RLHF 寻找最大化可测 reward 的 policy；LM 在后者中是 policy 而非自然数据分布模型。

**第 51 页。** InstructGPT 的 PPO objective 看起来很无害：reward model score、相对 SFT reference 的 token KL penalty，并可混 pretraining gradients；实际要协调 RM、RL policy、reference 与 PTX loss。

**第 52 页。** Stiennon《Learning to summarize from human feedback》给更多背景：先用 pairwise comparison 训练 reward model，再以 PPO fine-tune；KL penalty 防 policy 漂离 supervised initialization，value function/GAE 估 advantage，并详细说明训练超参。

**第 53 页。** PPO 概念演进：尝试一 policy gradient $\nabla E[R]=E[R\nabla\log p]$，方差太高；尝试二 TRPO，在 current policy 附近线性化并加 KL constraint；尝试三 PPO，对 importance ratio 在 $1\pm\epsilon$ clip。

**第 54 页。** 能否摆脱 PPO/on-policy RL？曾有多种合理方案：pair 上 SFT，chosen 前加 [GOOD]、rejected 前加 [BAD] control token；只训 preferred output；训 RM 后生成再训 preferred；让 RM 从1024个 outputs 选 best-of-N。

**第 55 页。** DPO 被称为“无泪 RLHF”：去掉 reward model 与 rollouts/outer loops 等 on-policy 部分；改为对 good log-loss 正向更新、对 bad 适当加权反向更新。图对比传统 RLHF 与 DPO pipeline。

**第 56 页。** DPO 从 RLHF objective 推导：最大化 reward 减 $\beta KL(\pi\|\pi_{ref})$。假定 policy space 包含所有 distributions，最优 $\pi_r(y|x)=\pi_{ref}(y|x)e^{r/\beta}/Z(x)$；反解 implied reward 为 $\beta\log(\pi_r/\pi_{ref})+\beta\log Z(x)$。讲义说明 Kimi-Think 也用此等价。

**第 57 页。** 用 Stiennon pairwise objective 优化 implied reward，代入上式得到 DPO loss。关键步骤：做 nonparametric assumption，把 policy 与 reward 闭式连接；用 policy 参数化 reward；用 supervised pair loss 优化 reward，从而优化 policy。概念上是该假设和替代参数化下对 pairwise rewards 做 MLE。

**第 58 页。** DPO update 可理解为提高 good、降低 bad，但梯度被 implied reward model 的 prediction error 缩放：当当前模型把 pair 排错时权重大，排对且 margin 大时权重小；括号中分别是 chosen/rejected log-prob gradients。

**第 59 页。** LLaMA/其他开放模型常把 DPO 与 expert iteration 合用：采 prompts、生成 K candidates、reward model/rejection sampling 选 preferred，SFT 后再 DPO；流程中还可能混 specialized/high-quality SFT data。

**第 60 页。** Variants 太多，Tülu 3 特别列两种：SimPO 不用 reference，以 length-normalized policy logprob 与 margin 构造 sigmoid；length-normalized DPO 把 chosen/rejected 的 log-ratio 各除 response length。

**第 61 页。** PPO 也能做到，而且有时更好。RL empirical work 的难点是结果高度依赖具体 setup；页面并列“Unpacking DPO and PPO”与 Tülu 3 的对比图/表，不支持一句“DPO 总胜 PPO”。

**第 62 页。** RLHF 要防 reward overoptimization/overfitting 与 mode collapse/entropy 下降。图中随 KL distance 加大，多个 proxy rewards 先升后降；calibration curve 也因 RLHF 改变。

**第 63 页。** 多种 RLHF optimizer 都会在某点后过拟合 reward。三图显示 human preference、noisy AlpacaFarm LM preference 会先提高后下降；single-prompt GPT-4 的 noiseless LM preference 持续上升，说明 reward noise/misspecification 是关键。

**第 64 页。** Mode collapse：RLHF 让模型默认不再是 calibration 良好的 probabilistic model。页面比较 RLHF 前后 MMLU 5-shot reliability、confidence histogram 与输出长度/类型分布，显示概率与多样性可被扭曲。

**第 65 页。** 总结：RLHF data collection 同样困难且有很多 confounders；RLHF algorithms 比 SFT 复杂，尤其 PPO；必须警惕过度优化 reward 的影响。

## Lecture 16：用可验证奖励训练推理

RLHF 的 reward model 是近似的人类偏好代理，优化过头会钻误差；RLVR（Reinforcement Learning from Verifiable Rewards）把问题收窄到有可靠 checker 的 math、code、formal proof 或 environment outcome，尽量让训练 reward 就是想要的结果。它不能解决所有对齐，却能让 rollout 规模化而不必为每条轨迹付人工标注。

### 1. 从 sequence reward 得到 token gradient

对 prompt $x$，policy 逐 token 生成 $y=(a_1,\ldots,a_T)$，整条序列只在末尾得到 $R(x,y)$。利用 log-derivative：

$$
\nabla_\theta J
=\nabla_\theta\mathbb E_{y\sim\pi_\theta}[R]
=\mathbb E\left[R\nabla_\theta\log\pi_\theta(y|x)\right]
=\mathbb E\left[R\sum_t\nabla_\theta\log\pi_\theta(a_t|s_t)\right].
$$

这解释了为什么一个 terminal reward 能训练所有 response tokens，也解释了 credit assignment 粗糙：每个 token 先得到同一 outcome signal。减去只依赖 state/prompt、不依赖当前 action 的 baseline $b(s)$ 不改变期望：

$$
\mathbb E_{a\sim\pi}[b(s)\nabla\log\pi(a|s)]
=b(s)\nabla\sum_a\pi(a|s)=0.
$$

Advantage $A=R-b$ 把“绝对得分”改成“比预期好多少”，显著降方差。PPO 用 value model $V_\psi(s_t)$ 学 baseline；GAE 定义 $\delta_t=r_t+\gamma V(s_{t+1})-V(s_t)$，再求 $\hat A_t=\sum_l(\gamma\lambda)^l\delta_{t+l}$。对只有末尾 reward 的一次性 LM bandit，$\gamma=\lambda=1$ 时本质是 terminal reward-to-go 减 value。

### 2. PPO：clipping 只限制 surrogate，不是硬限制 policy

Rollout 由 frozen old policy $\pi_{old}$ 生成，更新 policy 用 importance ratio：

$$
\rho_t(\theta)=\frac{\pi_\theta(a_t|s_t)}{\pi_{old}(a_t|s_t)},
$$

PPO surrogate 为：

$$
L^{clip}=\mathbb E_t\min\left(ho_tA_t,
\operatorname{clip}(\rho_t,1-\epsilon,1+\epsilon)A_t\right).
$$

<ImportanceClipLab />

当 $A>0$，ratio 超过 $1+\epsilon$ 后不再奖励继续放大 chosen action；当 $A<0$，ratio 低于 $1-\epsilon$ 后不再奖励继续压低。另一侧仍可能有 gradient，所以它不是把所有 ratio 简单 clamp，也不保证更新后的全局 KL 必在某阈值内。训练仍要监控 KL、clip fraction 与 ratio tails。

AlpacaFarm 式 LM PPO 外层先 rollout，再多 minibatch/epoch 优化。Reward shaping 通常在每 token 加相对 reference 的负 KL，在最后 response token 加完整 RM/verifier reward：

$$
r_t^{shaped}=-\beta\left(\log\pi_{old}(a_t|s_t)-\log\pi_{ref}(a_t|s_t)\right)
+\mathbf 1[t=T]R(x,y).
$$

有实现会在新 policy logp 低于 reference 时 clip 某一方向的 KL，防模型爆炸后 penalty 发散；这是经验稳定器，会改变理论 objective，应记录。PPO 还要同时训练 value、whiten rewards/advantages、mask padding、处理 truncation 与 EOS、同步 rollout weights，复杂度远超一条公式。

### 3. GRPO：去掉 critic，却加入了组内重加权

GRPO 对同一 prompt 采 $G$ 个 completions，reward 为 $r_1,\ldots,r_G$，标准 advantage：

$$
A_i=\frac{r_i-\bar r}{s_r+\varepsilon}.
$$

然后沿 response tokens 复用 $A_i$，使用 PPO-style old-policy ratio clipping 与 KL regularization。Online rollout 后立刻一次更新时，它接近 group-normalized REINFORCE；优点是无需 value model 显存/调参，代码可很短。

<AdvantageNormalizerLab />

组均值 subtraction 提供 prompt-specific difficulty baseline；若全对或全错，所有 $A_i=0$，没有信号。标准差 division **不是** 与 action 无关的 baseline，而是随机、reward-dependent scale：中等难度组若方差小会被放大；极易/极难组的权重关系也改变。严格 unbiased 的方案可对第 i 个样本使用 leave-one-out mean $b_i=\frac1{G-1}\sum_{j\ne i}r_j$，不除随机 std；包含自身的 mean 还带一个固定缩放，而 std 会引入更实质的 objective reweighting。

### 4. GRPO 的两种长度偏差与 Dr.GRPO 修正

标准实现常先对每条 response 的 token loss 除以 $|y_i|$，再对 group 平均。因为同一 $A_i$ 分给全部 tokens：

- 正 advantage 时，短正确答案每 token/序列更新更强，policy 偏爱较短正确 response；
- 负 advantage 时，长错误答案因除数更大而少受罚，错误集合中反而偏向长输出。

Dr.GRPO 去掉 group std normalization，并把 response-specific $1/|y_i|$ 改成统一长度尺度/按全部有效 tokens 聚合，目标是消除 question difficulty 与 response length 的隐式重加权。实验中 reward 相近，但标准 GRPO 的 output length 持续涨到约千 token，Dr.GRPO 约稳定在五百；错误答案尤其变短。修正不是“让梯度更大”，而是改变哪些 prompt/token 被当作一个等权样本。

还要区分真正能力增长与 metric artifact。R1-Zero 训练中 CoT 变长、出现自我检查“aha moment”；后续 Dr.GRPO 分析指出一部分长度增长来自 biased objective，且 base model 本来就会说“aha”。行为涌现的因果结论必须有 base/schedule/objective ablation。

### 5. DeepSeek-R1：最终 recipe 远多于 R1-Zero

R1-Zero 从 DeepSeek-V3 base 出发，data 未公开，只用 accuracy reward 与要求 thinking tags 的 format reward，GRPO 且不使用 process supervision；结果略低于当时 OpenAI o1。它的重要性是说明 outcome reward 可推动长 CoT、自检与数学/代码能力，不必预设 MCTS/PRM。

完整 R1 加了四段：

1. cold-start reasoning SFT，长 CoT 经 rejection/人工后处理，来源披露有限；
2. GRPO reasoning RL，加 language-consistency reward，抑制 CoT 中混语；
3. 两个 epoch SFT：约600K非可验证 reasoning（由 V3 judge）+200K通用 V3 SFT；
4. 再做 reasoning RL 与通用 non-verifiable RLHF，后者仍用 GRPO。

小数据也能 bootstrap：讲义对比约1K math/science prompts+Gemini/R1 long CoT 的 SFT，sample efficiency 很高。最终 R1 再生成约800K CoT traces，distill 到 Qwen2.5/Llama 等更小非 reasoning models；这说明 R1 的开放影响部分来自可蒸馏 data，而非只来自 RL algorithm。论文还诚实列出 PRM、MCTS unsuccessful attempts：process label 难且 reward hacking，token generation 下 tree search 的 value/search space 也难稳定。

### 6. Kimi k1.5：curriculum、reference reward 与显式长度控制

Kimi 与 R1 同期，也用 RL 超过 o1。Long-CoT recipe 是 dataset construction/difficulty filtering→long-CoT SFT→自有 policy-gradient RL。数据按 math topic 平衡，排除 multiple-choice/true-false 以减少 false positive，只留 model best-of-8 仍失败的题；SFT 细节很少，称为 prompt engineering，可能包含 distillation。

Kimi objective 带 reference regularization，论文也用类似 DPO 的 nonparametric 反解 reward，再以 squared loss surrogate 得到带 baseline 的 policy gradient。它没有同一 GRPO sequence-length normalization 问题，但另设 length reward 压缩 CoT：group 内越长 $\lambda$ 越负，正确答案被鼓励更短，错误答案被鼓励短于 rollout range 中心；只在训练后段打开，因为过早会损性能。

Curriculum 给题标 difficulty，从 easy 到 hard；采样概率正比 $1-\text{success rate}$，减少已经解决问题的重复。Code reward 用有 ground-truth solution 的问题生成更多 tests；math 用约800K samples 训练 CoT reward model 做 answer-equivalence，人工 spot check 中 classic RM 约84.4%、CoT RM 约98.5%。这仍是 learned verifier，需防等价判断误差。

### 7. Qwen 3：低数据 RLVR、thinking mode 与阶段回退

Qwen 3 先做 long-CoT cold start 和 reasoning RL，再 thinking-mode fusion、general RL，最后蒸馏给 lightweight models。题也按 best-of-n 难度筛：删无需 CoT 就答对的过易题、删与 validation 太近的题，并人工区分 CoT 真推理还是 guessing；GRPO reasoning RL 只用 **3,995 examples**，强调 selection 胜过盲目规模。

Thinking-mode fusion 把 thinking/non-thinking data 用 tags 混训，让用户控制是否展开；超预算时用特殊字符串触发 early-stop，然后要求模型给 final answer。Test-time thinking budget 增大通常提高 AIME/LiveCodeBench/GPQA，但曲线会饱和。阶段表也显示 general-purpose RLHF 后 math/STEM 会略降，提醒每阶段要做 capability regression，而不是只看综合榜。

### 8. Rollout 系统是算法的一部分

On-policy 意味慢速 autoregressive inference；training 与 serving 常用不同框架；长 CoT 让 batch ragged。典型系统包含 rollout workers、reward/verifier services、replay buffer、trainer workers、master/weight synchronization。训练后权重广播给 inference engine，会 invalidate CUDA graph/KV/prefix cache；广播太少则 policy stale、importance ratios 扩散。

### 1. CLIP：用 batch 内配对关系学习语义坐标系

给 batch 中 $B$ 对 $(I_i,T_i)$，image/text encoders 产生 L2-normalized $v_i,t_i\in\mathbb R^d$，温度 $\tau$ 的 logits：

$$
s_{ij}=\frac{v_i^\top t_j}{\tau}.
$$

CLIP 对每行做 image→text cross entropy、每列做 text→image cross entropy：

$$
\mathcal L_{CLIP}=\frac12\left[
-\frac1B\sum_i\log\frac{e^{s_{ii}}}{\sum_j e^{s_{ij}}}
-\frac1B\sum_j\log\frac{e^{s_{jj}}}{\sum_i e^{s_{ij}}}
\right].
$$

对角 pair 是 positive，其余 $B(B-1)$ 是 in-batch negatives。大 batch 带更多 negatives，却要求跨 devices all-gather embeddings 才能计算 global denominator；重复 caption/相似 image 还会产生 false negatives。

OpenAI CLIP 从500K queries 各找约20K image-text pairs，最终400M，但未公开；OpenCLIP 用 LAION-5B 复现，也用 CLIP 自身过滤，存在 feedback loop。原 preprocessing 把任意宽高图用 bicubic resize 到短边336，再 center crop 336²，边缘会丢；最佳是 ViT-L/14@336。Text encoder 是63M、12-layer GPT-2 Transformer，读取 BOS…EOS 后最高层 EOS activation。

Vision Transformer 把图像切 patches，flatten+linear projection，加 position embedding，再过 Transformer。CLIP 也试 ResNet-50 与 attention pooling——global-average activation 作为 query 做 QKV。Zero-shot classification 把类别名写 prompts，编码为 text prototypes，选相似度最高类；在 ImageNet 上胜过用1.2M labels 训练的 ResNet-50。直接让 image encoder autoregressively 预测 caption 更细，却比对比 ranking 计算效率低。

### 2. SigLIP：把全局多分类改成 pairwise 二分类

SigLIP 不要求“一行只有一个正确类别”的 softmax，而为每个 pair 定 $z_{ij}=+1$ 若 $i=j$、否则 $-1$，用 sigmoid logistic loss：

$$
\mathcal L_{SigLIP}=\frac1{B^2}\sum_{ij}
\log\left(1+\exp[-z_{ij}(\alpha v_i^\top t_j+b)]\right).
$$

各 pair 的 normalization 独立，distributed training 可分块计算/交换，不必让 loss 强绑定 global batch；小于16K batch 时优于 CLIP，实验把 batch 推到1M却发现约32K已足够。课程给出的资源对比是 CLIP 256 TPUv3×10天，SigLIP 32 TPUv4×5天，虽硬件不能只按卡数精确比，也显示 objective/communication 更高效。

WebLI 是十亿量级 internet image-text pairs，自动 OCR 抽 image text，只保最高质量10%，支持100 languages。OCR caption 能教文字识别，也会把错误 OCR 与隐私内容带入训练；language/domain retention 要分开评估。

### 3. LLaVA：连续视觉 token 如何接进 causal LM

原 LLaVA 组合 frozen CLIP ViT-L/14 与 Vicuna。若 vision encoder 输出 $H_v\in\mathbb R^{N_v\times d_v}$，linear projector $W\in\mathbb R^{d_v\times d_{LM}}$ 得：

$$
H_{img}=H_vW,
$$

再把 $N_v$ 个连续 image embeddings 插到 <code>&lt;image&gt;</code> 位置，与 text embeddings 一起过 causal LM。它们不是 vocabulary IDs，却作为同一 hidden sequence 的输入。

训练分两阶段：alignment 冻结 vision+LM、只训 $W$，学会把已有视觉语义旋到 LM space；instruction fine-tuning 冻结 vision、训练 projector+LM，学问答/对话。Data 从 MS COCO images、bounding boxes、Mechanical Turk captions 出发，让 GPT-4 仅看 captions/detected objects 合成 questions/conversations，再与原图配对，共158K。合成 teacher 没真正看图，可能写出 object list 支持不了的关系，必须以 image-grounded verifier/抽查过滤。

### 4. AnyRes 与 OneVision：高分辨率首先是 token 预算问题

CLIP 336² crop 对 OCR/diagram 不够。AnyRes 把原图选成 $a\times b$ tiles，每 tile 匹配 vision encoder resolution，分别编码再 concat，通常另加缩略全局图。若原图过大、tokens 超预算，再用 bilinear interpolation/downsample。Tile overlap、重复边界与二维顺序都影响模型。

LLaVA-OneVision 用 SigLIP 最后层前后 grid features、Qwen2-72B、2-layer MLP，统一 single image/multi-image/video：单图给高 resolution；多图每张 base resolution；video 每 frame 更低 resolution，目的是让三类输入产生近似 sequence length。训练以 quality over quantity、easy→hard 组织，大量 synthetic/task-specific data；单图 diagram/chart 可迁移多图，单图 OCR+多图关系可迁移 GUI agent，单图 circle visual prompt 可迁移 video。

视觉 token 数近似：

$$
N_v\approx n_{frames}\times n_{tiles/frame}\times
\frac{H_{enc}W_{enc}}{P^2\,c},
$$

$c$ 是 2×2 merge 等 compression。Attention/LM compute 随 text+visual sequence 增长，video 的低 information density 会淹没 text loss；不是 context window 装得下就值得全部保留。

### 5. Qwen-VL→Qwen2-VL→Qwen3-VL 的三次改造

**Qwen-VL。** OpenCLIP ViT-bigG/ViT-bigC 风格 14×14 patches；一层 cross-attention adaptor 加2D position，压成固定256 tokens；用 <code>&lt;img&gt;</code>、<code>&lt;box&gt;</code>、<code>&lt;ref&gt;</code> 表示模态/定位。Stage1 大量低质量 data，freeze LM、训 vision+adaptor；Stage2 高质量 task data、提高 resolution、全参数训练；Stage3 instruction data，freeze vision、训 adaptor+LM。

**Qwen2-VL。** 675M ViT 与 dynamic resolution。讲义口径是每224×224 region 由 ViT/14 编码，2×2 merge 后约66 visual tokens；video 2 frames/s、最多16,384 tokens。不同图产生不同 token 数，不再强压固定256。MRoPE 把同一 rotary dimensions 的 position coordinate 分给 time/height/width：text 三轴同步递增，image 的 t 固定而 h/w 网格变化，video 三轴都变。训练从 Qwen2 LM 与 DFN vision init 开始：vision-only→all parameters→instruction-following LM。

**Qwen3-VL。** Language backbone 覆盖 dense/MoE 到235B-A22B、256K context；vision 用 SigLIP-2。Interleaved MRoPE 不把频率段整块分成 tttt/wwww/hhhh，而交错成 t,w,h,t,w,h，让每个轴同时获得低/高频；video timestamp 另作为 explicit tokens，而不只藏 position embedding。DeepStack cross-layer fusion 把多层 vision features 注入 LM 多层，避免只在入口融合。

Video token 很长，若按所有 tokens 普通平均，会支配 batch；Qwen3-VL 用 square-root-normalized per-token loss 平衡 text/multimodal example。Pretraining 四阶段依次 adapter、全参8K、32K、256K；posttraining 用 long-CoT SFT、knowledge distillation、RL。课程结论是性能 SOTA、data work 很多但细节有限，architecture 改动小而可能关键，并靠规模放大。

### 6. 理解与生成需要不同 tokenizer

CLIP/SigLIP continuous embedding 被 text semantics 监督，适合理解，却不能被 LM 当离散类别逐 token 生成；通常另接 diffusion decoder。Chameleon 走另一极端：所有模态变 discrete IDs，统一 autoregressive modeling，可在文本中间理解/生成图片。

VQ-VAE encoder 得 latent $z_e$，对每个位置选最近 codebook vector：

$$
k=\arg\min_j\|z_e-e_j\|_2^2,\qquad z_q=e_k,
$$

decoder 从 $z_q$ 重建 pixels，用 reconstruction、codebook 与 commitment losses 学离散表。Chameleon 把512² image 编为1024 tokens，codebook size8192，并训练新的 BPE text tokenizer；image token 能预测，却在量化时损失细小文字/纹理。

Stage1 占80%：2.9T text tokens、1.5T text/image tokens、400B interleaved tokens；Stage2 占20%，一半 stage1 data、一半 high-quality。多模态训练不稳定，因为 text token entropy 较低、image token entropy 高，导致 norm growth/logit drift；QK normalization 与 z-loss（惩罚 log-partition 过大）帮助稳定。

以下对应 <code>lecture_17.py</code> 的全部讲授文本、架构、数据、训练阶段与示例。

**开场。** 第 17 讲主题是 multimodal models。此前 language model 做 text⇒text；现实世界是多模态。终极目标 omni model：输入任意模态组合以理解，输出任意模态组合以生成。今天 Transformer 很强，所以继续用；Transformer 说 token——离散或连续，每个 token 代表某种近似 semantic information unit——因此必须把一切转换为 tokens。Text 也曾在 tokenization 讲这样处理，但 non-text 更难。两个问题是怎样输入 non-text 如理解 image，怎样输出 non-text 如生成 audio。路线依次为 CLIP、SigLIP；LLaVA/OneVision、Qwen-VL/2/3；最后 Chameleon。

**CLIP 背景与方法。** CLIP 全称 Contrastive Language-Image Pretraining。过去 CV model 在 annotated images 上训练；问题是能否利用数量大得多的 image-caption pairs。方法在一个例如32,768的 image/text batch 中分别编码；每张 image 偏好自己的 aligned text 而非其他 texts，每段 text 也偏好自己的 aligned image；讲义展示 CLIP 双向 similarity matrix 与核心代码。

CLIP data 用500K search queries，每 query 约20K pairs，总400M image-text pairs，未发布。OpenCLIP 用 LAION-5B 复现，而 LAION 又用 CLIP 过滤。Processing 代码把任意 $W\times H$ image bicubic resize 到短边336，再 center crop 成336×336，因此会切边。

Vision encoder 试 ResNet-50 与 Vision Transformer；ViT 把 image 切 patches、线性 embedding、加 position、进 Transformer。Attention pooling 以 activation global average 当 query 做 QKV。最佳 ViT-L/14@336px：L 是 large，patch 14×14、3 channels，在336² images 训练。Text encoder 是63M/12-layer GPT-2 Transformer，编码 BOS…EOS，返回最高层 EOS activation。

Headline result 是 ImageNet zero-shot CLIP 胜过在1.2M ImageNet labels 训练的 ResNet-50。Ablation 发现直接从 image 预测 text 比 CLIP ranking 计算效率低得多。总结：image encoding 捕捉 noisy text 给出的 semantics；设计按 image classification 选择，未必细粒度；技术上需大 batch 与全 batch softmax。

**SigLIP。** Sigmoid Loss for Language Image Pre-Training。CLIP 把一个 text 对全部 images 作 multiclass；SigLIP 对每个 text-image pair 二分类“是否 aligned”，讲义展示 sigmoid code。

Data 是十亿量级 WebLI internet pairs；自动 OCR 抽 image text，只留最高质量10%，支持100 languages。效率对比：CLIP 用256 TPUv3十天，SigLIP 用32 TPUv4五天——TPUv4 此处标称 FLOP/s 还低于 TPUv3——明显更快。它把 batch size 与 loss 解耦，小于16K batch 优于 CLIP；实验最大1M，但32K已足够。图展示 data/embedding 在 device 间分块交换的 parallelism。

**LLaVA。** Large Language and Vision Assistant 的 vision encoder 是 CLIP，text decoder 是在 ShareGPT conversations 上 fine-tune LLaMA 的 Vicuna。MS COCO 有 bounding boxes 与 Mechanical Turk captions；把 captions/detected objects 给 GPT-4 生成 questions/conversations，再和原 images 配对，共158K，讲义展示一张街景的三类合成对话。

Model 用 CLIP ViT-L/14 编 image，一层 linear $W$ 投到 LM embedding space；Flamingo/Q-Former 更复杂。Stage1 alignment 冻 vision+LM，只训 W；Stage2 fine-tuning 冻 vision，训 W+LM。示例显示模型描述餐桌图片并回答细节。

**LLaVA-OneVision。** 它是 LLaVA1.5/LLaVA-Next 后版本，支持多图/video。Vision encoder 用 SigLIP 最后 Transformer layer 前后 grid features，text decoder Qwen2-72B，projector 2-layer MLP。

High resolution 对 OCR 重要；CLIP resize/crop 336² 丢信息。LLaVA1.5 引入 AnyRes：把 image 分 $a\times b$ 块、每块匹配 encoder resolution，编码后 concat；原图过高导致 tokens 太多时用 bilinear interpolation。图示全局缩略图、多个 tiles 与 newline/flatten 顺序。

统一 single image、multiple images、video 的目标是令不同 modalities 产生约同样长度：single image 用更高 resolution；multi-image 每张 base resolution；video 每 frame 更低 resolution。Data 哲学 quality over quantity，图表列 image/video/task sources；training 从 easier 到 harder 分阶段。

跨模态 transfer 例：single-image diagram/chart data 泛化 multi-image；single-image OCR 加 multi-image relational reasoning 泛化 GUI agents；single-image 中 circle visual prompting 泛化 video。总结：标准 VLM 是 vision encoder+projector+LM；大量工作在 synthetic/task-specific data curation；OneVision 开源 weights 与 data。

**Qwen-VL。** Vision encoder 是 OpenCLIP ViT-bigC、14×14 patches；adaptor 是一层 cross-attention，加入2D positional encoding，映射固定256 length；special tokens 有 <code>&lt;img&gt;</code>、<code>&lt;box&gt;</code>、<code>&lt;ref&gt;</code>。Stage1 大规模低质量 data，freeze LM、训 vision+adaptor；Stage2 更高质量 task-specific data、提高 resolution、全参训练；Stage3 instruction tuning，freeze vision、训 adaptor+LM。讲义展示定位、OCR、问答 examples。

**Qwen2-VL。** Visual encoder 升到675M ViT，关键是 dynamic resolution。每224×224 patch-region 以 ViT/14 编码，2×2 compression 后约66 tokens；video 每秒2 frames，最多16,384 tokens。MRoPE 图示 text、image、video 的 temporal/height/width position IDs。

初始化使用 Qwen2 LM 与 DFN vision encoder。训练类似 Qwen-VL：stage1 只训 visual encoder；stage2 全参；stage3 在 instruction-following data 训 LM。能力图覆盖 image question、video、OCR、chart、localization、agent interaction 等。

**Qwen3-VL。** Language model 是 Qwen3 dense/MoE，最大235B-A22B，long context 256K。Vision encoder 是与 SigLIP 同架构的 SigLIP-2。Interleaved MRoPE 把 temporal/width/height 分布到低频与高频，顺序 t,w,h 重复，而非先全 t、再全 w、再全 h。Video timestamp 用 separate tokens 显式给出，而不是塞在 position embedding。

它用 square-root-normalized per-token loss 平衡 text 与 multimodal data，避免很长 video examples 支配。Adapter 是 DeepStack cross-layer fusion，把 visual information 注入多个 layers。Pretraining 四阶段：adapter、全参8K、32K、256K；posttraining 是 long-CoT SFT、knowledge distillation、RL。结果图显示 SOTA。总结是大量 data work 但细节不多、少量可能重要的 architecture 改进、继续 scale up。

**Chameleon 与离散生成。** 之前 VLM 经 CLIP/SigLIP 编 image 再注入 LM，缺点是不能生成 image，需 diffusion。Chameleon 把一切映成 discrete tokens，能统一分析和生成；讲义展示文字与图像交错序列/生成例。

Vision encoder 的关键是映到可生成的 discrete tokens。VQ-VAE 将 image 映射到 discrete codebook，再 decode 回 image、最小化 reconstruction loss。512×512 image 编成1024 tokens，codebook size8192，同时训练新 BPE tokenizer。

Training stage1 占80%，含2.9T text tokens、1.5T text/image tokens、400B text/image interleaved；stage2 占20%，一半 stage1 data、一半 high-quality。Text token entropy 低、image token entropy 高，导致 norm growth 与 logit drift；修复用 QK norm、z-loss regularization。总结：统一 discrete autoregressive modeling 很优雅，但 discretization 丢信息、OCR 等性能不如连续方案，多模态一起训练很难。

**全讲总结。** Frontier model 预期 natively multimodal/omni。基础挑战是 non-text encoding；理解看 semantics，生成需要 finer details，两者需求不同；image/video information density 较低，要与 text balance 以保持稳定；现实强系统常用 continuous encoder+Transformer 做理解、diffusion 做 generation。

## 把三讲连起来

多模态模型同样需要 pretraining → instruction SFT → preference/RL，且数据和评测更脆弱：图文版权/隐私、OCR 噪声、空间幻觉、视觉 prompt injection、安全拒答。CS336 前 16 讲的 tokenizer、系统、数据、评测与对齐思维并没有失效，只是“token”不再全是文字。
