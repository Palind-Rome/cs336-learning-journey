## Lecture 9：Scaling law 在测什么？

假设有人给你一万张 B200 用一个月。A2 帮你搭训练系统，A4 帮你准备数据；真正昂贵的问题是：该训练多宽、多深、多少参数、多少 token、什么 optimizer？Scaling law 把“大模型上试错”改写为“小规模测量 → 拟合规律 → 外推决策”，但前提是明确它测的是哪一套 recipe。

### 1. 为什么 log-log 直线意味着可预测

数据幂律常写：

$$
L(D)=E+A D^{-\alpha}.
$$

E 是不可约 loss/entropy floor。若 E 可忽略：

$$
\log L \approx \log A-\alpha\log D.
$$

所以数据翻固定倍数 r，excess loss 每次乘固定比例 <code>r^(−α)</code>。这与线性坐标上“每增加一百万 token 固定改善”完全不同。拟合时若不处理 E，接近 floor 的曲线会弯，斜率被低估。

### 2. 从均值估计看“估计误差自然是多项式”

若 <code>x_i∼N(μ,σ²)</code>，样本均值的均方误差：

$$
\mathbb E(\hat\mu-\mu)^2=\frac{\sigma^2}{n},
\qquad
\log \operatorname{MSE}=-\log n+2\log\sigma.
$$

这是指数 α=1 的幂律。灵活非参数学习会更慢：在 d 维空间估光滑函数时，粗略 bias/variance 权衡可产生 <code>n^(−1/d)</code> 一类 dimension-dependent rate。神经 scaling 中 MT、speech、LM 的实测斜率明显不同于经典 1/n；“指数等于数据内在维度的倒数”是启发式解释，并非严密结论。

### 3. 数据不只看数量：mixture、shift 与重复

训练/目标 distribution 改变时，曲线可能主要改变 offset，也可能改变 slope 或 floor。多样数据能降低对某些目标的 distribution shift，但“用小模型挑最优 mixture”实践很难：小模型可能偏好易学来源，大模型才利用高难领域；tokenizer 和过滤也改变每 token 信息量。

有限 unique data 下，重复 token 的边际价值低于新 token。可把原始 D 替换为 effective data D′，它依赖 unique tokens Ud 与 repetition Rd，并在高重复区饱和。由此得到重要结论：最佳 data selection 应随训练规模自适应；小预算可以反复高质量核心集，大预算必须扩大多样性。Scaling curve 是在当前方法族下观察到的下界/frontier，换更好算法总能向下移动；盲目外推会“break”。

### 4. 架构、optimizer 与 shape 也可以做 scaling experiment

不要花数千万美元直接训练“LSTM 版 GPT-3”才比较 Transformer。对每个候选 recipe 在多个小规模测点，比较 fitted curves 与目标规模交点。Hestness、Kaplan、Tay 等结果表明 optimizer、architecture、depth 等影响在一定条件下可提前预测：

- 1→2 layers 收益巨大，再加 depth 在小参数区边际递减；
- 同总 N 时 embedding parameters 与 repeated block parameters 的“价值”不同；
- MoE total parameter 与 active parameter 的 scaling 也不同；
- Adam vs SGD 可能有不同 offset/slope；
- aspect ratio 是否随规模变化要由跨规模 sweep 判断，不能只固定一次。

每条 curve 必须公平调 schedule/learning rate/initialization；standard parameterization 下最佳 LR 会随 width 漂移，μP 一类 scale-aware recipe 是超参迁移的前提。

### 5. Critical batch 不是一个模型常数

对固定目标 loss，扫 batch B，记录到达目标所需 optimizer steps S(B) 与 examples E(B)=B·S(B)。小 B 时增 batch 几乎等比例减 steps；大 B 后 gradient noise 已被平均，再加 batch 只浪费 examples。临界点常通过拟合 Smin 与 Emin 的 tradeoff 曲线，选 steps/sample 两侧相对均衡的位置，约在各自最优的 2 倍范围。

目标 loss 越低、训练越后期，gradient signal-to-noise 常变化，critical batch 越大。因此“batch 4M”不能脱离训练阶段；DP scaling 必须同时看 hardware efficiency 与 sample efficiency。

### 6. N 与 D 的联合律

经典形式：

$$
L(N,D)=E+\frac{A}{N^\alpha}+\frac{B}{D^\beta},
\qquad C\approx6ND.
$$

固定 C 后 D=C/(6N)，loss 随 N 通常先降后升：模型太小受 capacity 限制，模型太大则 token 太少、data term 主导。最优条件由两项边际收益平衡，得到：

$$
N_{opt}\propto C^{\frac{\beta}{\alpha+\beta}},
\qquad
D_{opt}\propto C^{\frac{\alpha}{\alpha+\beta}}.
$$

Kaplan 早期报告约 N∝C^0.73、D∝C^0.27，意味着规模越大 token/parameter 反而下降；Chinchilla 发现应更均衡地扩数据与参数，粗略经验约 20 tokens/parameter。差异不是小数位争论，而会把巨额预算分配到完全不同模型。

### 7. Chinchilla 三种方法与失败模式

1. **训练曲线包络（minimum over runs）**：多种 N 的 loss-vs-compute curves 联合取每个 C 的最低点，再拟合下包络幂律。
2. **IsoFLOP**：对一组 C，运行多个 N（D 由 6ND=C 决定），每条 U 型曲线取最小 N，再拟合 <code>N_opt(C)</code> 与 <code>D_opt(C)</code>。
3. **joint parametric fit**：在 N×D grid 上拟合 <code>E+A/N^α+B/D^β</code>，用全部数据。

方法 1/2 直观且对函数形式依赖较弱，但必须让每个预算的 sweep 包住谷底；全在谷底一侧就只能给边界。方法 3 数据利用高，却对参数计数、loss 形式、outliers、优化器收敛和数值优化敏感。后续数据取证指出原 Chinchilla 方法 3 很可能有拟合问题，重拟合结果更接近方法 1/2。

Kaplan/Chinchilla 差异还来自：是否排除最后/embedding layer parameters、小预算 warmup 过长，以及 total vs non-embedding parameter 的小非线性。报告必须写清参数口径、schedule 和 fit range。

### 8. Training-optimal 不等于 deployment-optimal

Chinchilla 优化固定 training compute 下的 loss；真实产品还会为每次请求反复支付 inference。若预计生成大量 tokens，值得预先训练一个更小模型更久，即“overtrain”相对 compute-optimal 配方：

| 模型 | 约 tokens / parameter |
| --- | ---: |
| GPT-3 | 2 |
| Chinchilla | 20 |
| Llama 65B | 22 |
| Llama 2 70B | 29 |
| Mistral 7B | 110 |
| Llama 3 70B | 215 |

选择应最小化总生命周期成本：

$$
C_{total}=C_{train}+Q\cdot C_{inference/request},
$$

Q 越大，前期多训小模型越划算。IsoFLOP 也用于 diffusion、MoE 等领域，因为流程简单、frontier 清晰。

## Lecture 10：推理为何与训练不同

训练只支付一次，推理会在聊天、代码补全、agent、批处理、评测与 RL rollout 中反复支付。训练已知整段 token，可以沿 sequence 并行；自回归生成必须等上一个 token，工作负载从规整 dense tensor 变成不断到达/结束的 ragged requests。

### 1. 三类 serving 指标不能互相替代

- **TTFT**：交互请求等多久看到第一个 token；
- **单请求 latency / TPOT**：之后 token 出现多快；
- **throughput**：全系统每秒为所有请求产生多少 tokens。

小 batch 往往 TTFT/TPOT 好但权重读取无法摊薄、吞吐低；大 batch 吞吐高，但排队、KV cache 和每步 memory IO 增加，单请求更慢。报告还应包含 request/s、end-to-end latency 分位数、prompt/output 长度分布、显存与硬件，不可只给一个 tokens/s。

常用引擎包括 [vLLM](https://github.com/vllm-project/vllm)（PagedAttention）、[SGLang](https://sgl-project.github.io/)（RadixAttention/prefix workloads）、[TensorRT-LLM](https://nvidia.github.io/TensorRT-LLM/overview.html) 与 [llama.cpp](https://github.com/ggml-org/llama.cpp)。

### 2. 一个 matmul 已经解释 decode 为何 memory-bound

对 <code>X[B,D] @ W[D,F]</code>、bf16：

$$
FLOPs=2BDF,
$$

$$
bytes=2BD+2DF+2BF.
$$

当 B≪D,F 时，算术强度约 B FLOPs/byte。H100 dense bf16 峰值约 989TFLOP/s、HBM 3.35TB/s，平衡点约 295 FLOPs/byte；因此 B=1 的 matrix-vector intensity≈1，必须读巨大 W 却只使用一次，极度 memory-bound。

### 3. Prefill 与 decode 的完整算术强度

若每层 gated MLP 有 Wup/Wgate/Wdown，输入 <code>[B,T,D]</code>：

$$
F_{MLP}=6BTDF,
$$

$$
bytes_{MLP}=4BTD+4BTF+6DF,
$$

在 BT≪D,F 时 <code>I_MLP≈BT</code>。prefill 的 T 是 prompt 长度，很容易 compute-bound；decode 每次 T=1，只能依赖 concurrent requests B 提高复用。

FlashAttention 下，读 Q[B,T,D] 与历史 K/V[B,S,D]，两个 matmul：

$$
F_{attn}=4BSTD,qquad
bytes_{attn}=4BSD+4BTD,
$$

$$
I_{attn}=\frac{ST}{S+T}.
$$

prefill T=S 时强度 S/2；decode T=1 时 <code>S/(S+1)&lt;1</code>。注意 attention intensity 中 B 抵消：不同请求有不同 KV cache，batching 不能像共享 MLP weights 那样复用它。结论表：

| 阶段 | MLP intensity | attention intensity |
| --- | ---: | ---: |
| prefill | B·S | S/2 |
| decode | B | &lt; 1 |

### 4. KV cache 与理论 latency 上界

若 K 是 KV heads、H 是 head dim、L layers、S history、bf16：

$$
M_{KV/sequence}=S\cdot K H\cdot L\cdot2_{K,V}\cdot2_{bytes}.
$$

总 memory 是 parameter bytes 加 B 倍 KV。讲义用 memory-bound 极限：

$$
latency/token\approx\frac{M_{params}+B M_{KV/seq}}{HBM\ bandwidth},
$$

$$
throughput\approx\frac{B}{latency}.
$$

它假设 compute/IO 完美 overlap、忽略 kernel/通信，是真实性能的乐观上界。Llama 2 13B 示例配置：S1024、D5120、F13824、40 query/40 KV heads、head dim128、40 layers、vocab32k、H100 3.35TB/s。B 从 1→64→256 时 latency 变差、throughput 变好，但 B256 超 80GB，且吞吐边际递减。复制 M 份模型保持单副本 latency、总吞吐约乘 M；模型/KV 分片则更复杂。

TTFT 主要是 prefill，应使用较小 prefill batch/良好调度；decode 为摊权重可用较大 batch。现代引擎常分开调度两阶段，避免长 prefill 阻塞 decode。

### 5. 减 KV：四个可独立调的维度

**Heads。** MHA 为 K=N，MQA 为 K=1，GQA 介于中间。N query heads 分成 K 组，每组共享一对 K/V；cache 约缩 N/K。讲义把 Llama2-13B 从 K40 改 K8 后，B64 cache 大减且可容纳更大 batch；质量需用论文消融确认。

**Latent width。** MLA 不保存 <code>W_Kh,W_Vh</code> 的全 N·H 维，而存 <code>c=W_ch</code>，用时上投影。DeepSeek V2 示例从 16384 维压至 512；RoPE 与投影合并冲突，另留 64 维 decoupled RoPE，总 576。论文表中 MLA 可达到不低于 MHA 的质量，但结论依赖完整 recipe/kernel。

**Layers。** CLA 像 GQA 跨 heads 共享那样，跨相邻 layers 共享 K/V，改善 accuracy-cache Pareto frontier。

**Tokens。** sliding-window/local attention 只留最近 W，cache 不随总 sequence 无限长；经过 L 层有效 receptive field 可扩大，但精确长程能力可能损失，常与 global layers 交错。DeepSeek V4 讲义还列 CSA（每 m tokens 压成 1）、DSA（选 top-k）、HCA（更强压缩）。linear attention/SSM 则用固定状态替代 cache。

### 6. Quantization：scale 与 zero point 的机械含义

仿射量化：

$$
q=\operatorname{round}(x/s)+z,qquad
\hat x=(q-z)s.
$$

s 决定步距，z 让整数 0 对齐实数零附近。fp32 4 bytes、bf16 2、fp8/int8 1、int4 0.5；低 bytes 直接改善 memory-bound decode，但 dequantization、scale load、outlier path 与硬件 kernel 决定是否实现理论 speedup。

QAT 在 training forward 模拟量化误差，权重可适应但成本大。PTQ 用 calibration data 按 tensor/channel/group 估 scale；GPTQ 用 Hessian/二阶信息，量化一部分时调整其余权重补偿误差。AWQ 观察少量 activation channels 很大，与它们相乘的 weights 更重要，据 activation 保护约 0.1–1% weights/通道；论文报告 fp16→int3 约四倍低 memory、3.2× speedup。

### 7. Pruning + distillation：先删，再修

结构化 pruning 按小 calibration set 衡量 layer/head/hidden dimension 重要性，删除不重要结构得到真正更小 dense shape，再把原模型知识蒸馏回来。讲义引用 NVIDIA 工作：用 1024 calibration samples 选结构，剪枝后 distill。非结构化零值若没有 sparse kernel，参数数虽少却未必更快。

### 8. Speculative sampling 为什么仍精确

draft p 快速提出若干 tokens，target q 用一次 prefill-like 并行 forward 给所有候选位置打分。每个 draft token x 以：

$$
a(x)=\min\left(1,\frac{q(x)}{p(x)}\right)
$$

接受；拒绝后从 residual <code>[q−p]_+</code> 归一化采样，并保证至少 target 产一个 token。这是 modified rejection sampling，最终边缘分布严格为 q，不是近似。

二词例中若 $p(A)>q(A)$，则 $p(B)<q(B)$，residual 全在 B：

$$
P(output=A)=p(A)\frac{q(A)}{p(A)}=q(A),
$$

$$
P(output=B)=p(B)+p(A)\left(1-\frac{q(A)}{p(A)}\right)=q(B).
$$

速度取决于 draft/target 成本比、一次提议长度和接受率。常见 70B target/8B draft 或 8B/1B；distill draft 提高贴近度。Medusa 并行多 token heads，EAGLE 使用 target high-level features。若只贪心验证，也可加速 greedy decode，但“精确目标采样”的证明对应完整接受/残差算法。

### 9. Continuous batching + paging 处理 ragged traffic

静态 batch 要等最慢请求完成，早结束 slots 空闲，新请求也只能等。Orca 的 iteration-level scheduling 每个 decode step 后移出完成项、加入新项。不同 lengths 时：

- attention 按 sequence boundary 独立处理，不能让请求互相注意；
- norm/MLP 等 non-attention 把所有当前 tokens 拼成 <code>[sum lengths,H]</code> 批量算。

旧 KV allocator 按最大 prompt+response 连续预留，短回答留下 internal fragmentation，不同 allocation 间还有 external fragmentation。PagedAttention 把逻辑 KV sequence 映射到不连续固定 blocks；block table 像页表。相同 system prompt 或一个 prompt 多采样可共享 physical blocks，分叉时 block-level copy-on-write。vLLM 还融合 block read+attention、使用 FlashAttention/FlashDecoding 与 CUDA graphs 降 launch overhead。

**入口与全局路线。** 标题 inference。先理解 workload：生态、Transformer 记号、算术强度、推理强度、吞吐/延迟。随后研究有损 shortcut：减 KV、量化、剪枝；目标是减复杂度而不伤 accuracy。从零 recipe 是定义更快 architecture 并训练；distillation recipe 是定义更快 architecture、用不同 architecture 的原模型初始化/迁移，再蒸馏修复。无损但需检查的 shortcut 是 speculative sampling。动态 traffic 的难点是请求到达时间不同、prefix 可共享、lengths 不同；用 continuous batching 与 paged attention。总结推理服务真实使用/评测/RL，特性 memory-bound 且动态，技术包括新架构、量化、剪枝/蒸馏、speculation、paging。

**生态与指标。** 推理出现于 chat/code/agents/batch processing、instruction-following evaluation、RL 多次生成后评分。训练一次，推理重复；讲义引用 OpenAI 每日约 8.6T tokens、DeepSeek V4 训练 32T 作数量对比。聊天 token 给人看，人是瓶颈；agent 的 query→internal trace→human output 可产生无界 tokens，每 token 都是 compute。提供闭源模型的 OpenAI/Anthropic/Google，与开放权重服务 Together/Fireworks/Baseten/DeepInfra/Groq/Cerebras 都重视推理。开源软件为 vLLM、SGLang、TensorRT-LLM、llama.cpp。快的定义是 TTFT、单 query seconds/token latency、多 query tokens/s throughput。训练可沿已知 sequence 并行，生成必须串行，难以满算力。

**记号与单 matmul。** B batch、T sequence、D model、H head dim；contracting dimension 在两 operands 出现而在输出消失，batching dimension 两边和输出都保留。约定 F=4D、D=NH、GQA 中 N=KG、训练 S=T。X[B,D]×W[D,F] 读 2BD+2DF、写 2BF bytes，做 2BDF FLOPs，强度在 B≪D,F 时为 B。H100 平衡点 989e12/3.35e12≈295；强度大于它 compute-bound，小于它 memory-bound。B=1 matrix-vector 读 DF 权重只做 2DF FLOPs，正是 inference。

**KV cache 与两阶段。** naive 每生成 token 都把历史重跑，单次 attention O(T²)，生成 T tokens 累积 O(T³)。prefix 工作可共享，所以把每 sequence/token/layer/KV head 的 H 向量存 HBM。Prefill 对 prompt 并行，generation 串行。S 是 conditioning tokens，T 是本次要 logits 的 tokens。

Gated MLP 的逐步账本：读 X[B,T,D]；读三矩阵 Wup/Wgate/Wdown；分别算 U、G、GeLU(G)⊙U 后 down；总 6BTDF FLOPs、4BTD+4BTF+6DF bytes，BT 远小于 D/F 时强度 BT。prefill 大 B·T 容易计算受限；generation T=1 且 interactive B 不可预测。Attention 读 Q[B,T,D]、K/V[B,S,D]，QK 和 PV 共 4BSTD FLOPs，读写 4BSD+4BTD bytes，强度 ST/(S+T)。prefill T=S 得 S/2，generation T=1 小于 1；B 抵消，因为 MLP weights 跨请求共享，KV 不共享。总结即正文表格。

**性能统计代码。** 参数公式包含 tied/untied 两端 embedding 2VD、每层三 FFN 矩阵 3DFL、query/output 与 KV projections <code>(2DNH+2DKH)L</code>。bf16 parameter size 为 2P。每序列 KV 为 S·KH·L·2(K/V)·2 bytes；总 memory 为 B×KV+parameters。memory-bound latency 用总 memory/bandwidth，throughput 为 B/latency。dataclass 可对符号统计 substitute。

Llama2-13B 配置 S1024、D5120、F13824、N=K=40、H128、L40、V32000、bandwidth3.35e12。假设 compute/communication 完美 overlap 且无 overhead，B1 得单请求理论值；B64 latency 更差、throughput 更好；B256 更甚但超 80GB 且边际递减。增 batch 使 O(B) KV 读写增加，却摊 parameter read。小 batch latency 好/throughput 差，大 batch相反。复制 M model instances 保 latency、吞吐乘 M；分片 model/KV 更难。TTFT 近似 prefill，prefill 小 batch；generation 大 batch提吞吐。

**减 KV。** GQA 有 N query、K KV heads，每 KV 与 N/K queries；MHA K=N、MQA K=1。cache 缩 N/K，memory-bound 因而提速。讲义比较 K40/B64、K8/B64、K8/B256：GQA 单步 cache 改变使配置可装并提高吞吐，但 latency/throughput tradeoff仍在，并需 accuracy 表。MLA 存低维 c=Wch，用时上投影 K/V；DeepSeek V2 从 NH=16384 到 C512，再加 RoPE64=576。论文表先显示 MHA 质量高于 GQA，另表显示 MLA 略高于 MHA且更便宜。CLA 跨 layers 共享 KV，改善 accuracy-cache Pareto。Local/sliding attention 只看附近，effective context 随 layers 线性扩，cache 可不随全长，但可能伤 accuracy，故交错 global。DeepSeek V4 支持 1M context，讲义列 CSA 每 m 压 1、DSA top-k、HCA 更强压缩。总结为低维 cache、局部截断，以及 linear/SSM/diffusion 等其他方向。

**量化。** 例 x=5.2342、scale0.1、zero point4，quantize 为 round(x/scale)+z，dequantize 为 (q−z)scale。fp32 4 bytes 训练 state，bf16 2 是 inference 默认，H100 e4m3 fp8 1 byte 范围约 [−240,240] 并可冒险训练，int8 1 byte [−128,127] 更便宜但精度低且推理用，int4 0.5 byte [−8,7] 更便宜更不准。QAT forward 模拟误差，适配好但需昂贵训练。PTQ 便宜，用 sample data 定 layer/tensor scales；GPTQ 用 Hessian 更新未量化 weights 补误差。AWQ 发现大 activation channels 对应 weights 更重要，按 activation 保留 0.1–1% 高精度；论文示例 fp16→int3 低四倍 memory、3.2× speedup。

**剪枝。** 直接删除昂贵模型部分后修复。NVIDIA 论文用 1024 calibration samples 识别重要 layer/head/hidden dimension，删不重要结构，随后把原模型 distill 到 pruned model，并报告结果。

**Speculation。** Prefill 并行且也给 probabilities，generation 串行；checking 比 generation 快。便宜 draft p 猜约四 tokens，target q 一次并行评分并按 modified rejection sampling 接受；算法保证 exact q sample，且至少产一个候选。二词 A/B 证明：若 p(A)>q(A)，residual max(q−p,0) 只在 B；A 输出概率 p(A)q(A)/p(A)=q(A)，B 为 p(B)+p(A)(1−q(A)/p(A))=q(B)。实践 70B/8B 或 8B/1B target/draft，并 distill draft；Medusa 并行多 tokens，EAGLE 使用 target high-level features。总结是数学保证 exact、利用 checking/generation asymmetry，draft training 仍有创新空间。

**动态 batching。** Orca 指出训练是 B×S dense tokens，推理到达/结束不同时是 ragged array。iteration-level scheduling 每步 decode，并随时加入新请求，不等整批结束。不同 lengths 时 selective batching：attention 每 sequence 独立，non-attention 把 [3,H]、[9,H]、[5,H] 等 concat 为 [17,H]。

**PagedAttention。** 旧方法请求到达即为 prompt+最大 response 预留连续 KV，实际短输出造成 internal fragmentation，sections 间有 external fragmentation。PagedAttention 像 OS paging，把一个 sequence KV 切成不连续 blocks。多个请求可共享 system prompt，或同 prompt 多 responses（如 program synthesis）；prefix blocks 共享，写时 block-level copy-on-write。vLLM 还融合 block read+attention，采用 FlashAttention/FlashDecoding 与 CUDA graphs 降 launch overhead。总结是借 paging 管理动态 workload memory。

### 2. MiniCPM：μP 与 WSD 怎样省掉重复训练

MiniCPM 的做法是用 μP 稳定初始化和学习率，固定模型 aspect ratio，再整体放大。公开 sweep 使用约 9M、30M、70M、0.1B、0.17B、0.2B、0.5B 等尺度；最大的 proxy 与最终约 2.4B 模型仍有约 5 倍差距，所以迁移不是同尺度插值。

它分别验证三件事：

- μP 预言最优 LR 应大致不随宽度移动；因此画不同模型的完整 LR–loss 曲线，而不是只报一个点。
- batch sweep 把纵向一列 checkpoints 视为“固定 batch、训练数据逐渐增加”的一条轨迹；对每个目标 loss/数据量找红色 lower envelope，得到最佳 batch 随 loss 降低而多项式增大的关系。
- N/D 取舍不能拿 cosine run 的中途 checkpoint 冒充小数据完整训练。早停点还处在高 LR 阶段，而“从头只训练这么多 token”的 run 已经走完 decay，两者不是同一目标函数。

WSD 解决第三点。设总预算 D、衰减占比 ρ≈10%，schedule 可写成：

$$
\eta(t)=
\begin{cases}
\eta_{\max}\,t/T_w,&t<T_w,\\
\eta_{\max},&T_w\le t<(1-\rho)D,\\
\eta_{\max}\,g\!\left(\frac{t-(1-\rho)D}{\rho D}\right),&t\ge(1-\rho)D.
\end{cases}
$$

stable 阶段虽然 loss 降得比 cosine 慢，但在短 decay 中会迅速下降。更重要的是，可以保存 stable checkpoint，在不同 token 位置分别分叉一个约 10% decay；这样一个长主干就生成多个“确实完成衰减”的终点，避免为每个 D 从头重跑。它不是免费午餐：分叉点之前的 optimizer state、数据顺序和 LR 必须可复现，比较时也只能比较各自 decay 后的 loss。

MiniCPM 同时使用 Chinchilla method 1（各 compute 下的 lower envelope）与 method 3（联合拟合）。其公开拟合给出很高的 data/model ratio，意味着所测范围中额外数据的边际收益下降较慢；这只是该数据与 recipe 下的结果，不能直接搬成普适的“每参数固定 token 数”。

### 3. DeepSeek 与其他公开 recipe：相同名词，不同证据强度

DeepSeek 没用 μP，而是在小尺度上直接做 batch×LR grid search；把 loss 在最小值 0.25% 内的点视作 near-optimal，再拟合其随 compute 的变化。batch 趋势较清晰，讲义认为 LR 拟合显得可疑，提醒我们“论文画出直线”不等于残差足够小。为便宜地做 Chinchilla 分析，DeepSeek 采用快速 warmup 加两个各约 10% 的 decay 段，效果大致匹配 cosine；然后用 method 2 的 IsoFLOP 谷底拟合 N_opt 与 D_opt，最终 7B/67B loss 通常被曲线准确预测。

| MiniCPM | μP、LR、batch、WSD、method 1/3 | 可把稳定主干复用成多预算终点 | 高 data ratio 依赖其数据与参数计数口径 |
| DeepSeek LLM | LR/batch 网格、WSD 式 schedule、IsoFLOP | 显式拟合而非假设超参完全不变 | near-optimal 筛选会掩盖 basin 宽度，LR 趋势较弱 |
| Qwen 2.5/3 | batch 与 hyperparameter scaling | 大模型团队也在单独拟合 optimizer 超参 | 公开细节少，难以独立复现误差 |
| Kimi K2 | MoE sparsity scaling | active/total parameters 是另一条尺度轴 | dense N/D 定律不能直接替代 sparsity sweep |
| Hunyuan-Large | MoE active parameters 的 IsoFLOP | 报告最优数据:active-param 约 96:1 | active N 不等于总参数 N，训练/服务成本口径不同 |
| Llama 3 | IsoFLOP 与 compute→downstream | 报告约 39:1，并验证下游能力随 compute | 预训练 loss 最优与每个 downstream 最优未必一致 |
| MiniMax-01 | architecture 决策与 method 1 | attention 变体也可纳入 scaling 决策 | 架构 family 变了，不能只按参数量归一化 |

因此读工业报告要记下：预测目标、训练 FLOPs 口径、N 是总参数还是 active 参数、D 是否去重、数据质量是否恒定、模型是否 shape-similar、是否展示发散/边界点，以及最大 proxy 到目标模型的外推倍数。

### 4. Optimizer 也有 scaling law

StepFun 的研究把 LR 与 batch 当二维曲面做经验网格。对固定模型/数据，pretraining loss 在这两个方向通常形成可识别的凸形 basin，因而可以找 minimizer。联合改变 N 与 D 后，其观察是：最佳 batch 主要依赖 dataset size；固定模型而增加 D 时最佳 LR 可能上升，但这一趋势换成 WSD 后更脆弱。相似结构在 MoE 与其他数据集上出现，不过“出现相似等高线”仍不等于同一幂律参数。

比较 optimizer 时最常见的两种错误是：

1. 用 AdamW 的 LR/weight decay 直接跑 Lion、Muon 等方法，然后把调参不足当算法差异；讲义示例中 Lion 的最佳 weight decay 可到约 0.6，而 AdamW 使用另一组最优值。
2. 只在一个小模型、一个 Chinchilla ratio 比 loss。某 optimizer 相对 AdamW 的 speedup 可能随 130M→1.2B 明显收缩，也可能在不同 data/model ratio 下换序。

所以算法对比应至少报告 wall-clock/compute 对齐曲线、各自调过的超参、多个模型尺度和多个 D/N。一个看似良好的 IsoFLOP 外推也可能在更大预算突然爆炸；讲义引用的 Delphi 案例最终通过更谨慎的 AdamC、LR 随 batch 的平方根缩放，以及参数化/optimizer 调整修复。

### 5. Muon：为什么“把矩阵更新正交化”会改变尺度

Muon 专门处理隐藏层的 matrix-valued parameters。先由动量形成更新矩阵 $B_t$，若其 SVD 为 $B_t=USV^\top$，Newton–Schulz 迭代近似得到：

$$
\operatorname{msign}(B_t)\approx UV^\top.
$$

这会压平奇异值，而不是让最大奇异方向独占更新；embedding、bias、norm gain 等 vector-like 参数通常仍交给 AdamW。Muon 在 nanoGPT speedrun、专门 scaling study 与 Kimi K2 级别训练中都能工作，但“能在大尺度稳定运行”与“相对 AdamW 的固定倍率收益可外推”是两件事。必须把正交化开销、矩阵形状、分布式通信和各自最佳 LR 都计入比较。

### 6. μP 推导：控制的不是参数大小，而是函数变化

对第 $l$ 层宽度 $n_l$ 的深线性网络：

$$
h_l=W_lh_{l-1},\qquad
W_l\sim\mathcal N(0,\sigma_l^2I_{n_l\times n_{l-1}}).
$$

“baby μP”提出两个尺度条件：A1，初始化时每个 activation coordinate 为 $\Theta(1)$，所以 $\lVert h_l\rVert_2=\Theta(\sqrt{n_l})$；A2，一步梯度更新造成的 activation change 也应为 $\Theta(1)$ 每坐标，即范数 $\Theta(\sqrt{n_l})$。太小会进入 lazy/kernel regime，太大会随增宽发散。

由随机矩阵集中性，谱范数近似：

$$
\lVert W_l\rVert_*\to \sigma_l(\sqrt{n_{l-1}}+\sqrt{n_l}).
$$

选取

$$
\sigma_l=\frac{\sqrt{n_l}}{\sqrt{n_{l-1}}}
(\sqrt{n_l}+\sqrt{n_{l-1}})^{-1}
=\Theta\!\left(\frac1{\sqrt{n_{l-1}}}
\min\!\left(1,\sqrt{\frac{n_l}{n_{l-1}}}\right)\right),
$$

若归纳假设 $\lVert h_{l-1}\rVert_2=\Theta(\sqrt{n_{l-1}})$，就得到 $\lVert h_l\rVert_2=\sqrt{n_l}+o(\sqrt{n_l})$。讲义提醒这里把“≈”当上界，是偏 worst-case 的推导，不应误读成有限宽网络的精确等式。

对 SGD，线性层一步更新是 rank-one outer product：

$$
\Delta W_l=-\eta_l(\nabla_{h_l}\ell)h_{l-1}^{\top},
$$

且

$$
\Delta h_l=W_l\Delta h_{l-1}+\Delta W_l(h_{l-1}+\Delta h_{l-1}).
$$

若领先项不抵消，要让 $\Delta W_lh_{l-1}$ 与目标 $\Theta(\sqrt{n_l})$ 同阶，就需

$$
\lVert\Delta W_l\rVert_*=
\Theta\!\left(\sqrt{\frac{n_l}{n_{l-1}}}\right).
$$

再假设单步 loss change 为 $O(1)$，由 $\Delta\ell\approx\langle\Delta W_l,\nabla_{W_l}\ell\rangle$ 可推出 SGD 的

$$
\eta_l=\Theta\!\left(\frac{n_l}{n_{l-1}}\right).
$$

Adam 把逐坐标梯度大小归一化，更新尺度不同，对应讲义简化结果约为 $1/n_{l-1}$。因此 μP 不是“所有层统一把 LR 除以宽度”：embedding、attention 矩阵、MLP 输入/输出矩阵、LM head 分属不同参数族，各自还涉及 init variance、output multiplier 与 attention scale。

### 7. μP 迁移边界必须用 sweep 证明

最直接的验证是对宽度 128、512、2048 分别扫 base LR。若同一列 LR 都接近各自谷底，才叫 LR transfer；只比较一个恰好没炸的点不够。课程引用的复现实验显示 μP 总体比 SP 稳定，但现代 Transformer 偏离理想推导：SwiGLU/squared ReLU、不同 batch、zero-init attention、RMSNorm gains、Lion 和 regularization 都要逐项消融。

- projection bias 在所示实验中仍能 transfer。
- learnable RMSNorm gain 是 vector-like 参数，用相同 base LR 会破坏迁移；去掉 gain 几乎不损性能，或必须为它单独定 scaling rule。
- 只依赖 gradient sign 的 Lion 在表中没有 transfer。
- 强 decoupled weight decay（0.1）是明显失败项之一，因为其更新不来自 loss gradient，不能自动满足 A2。

**第 46 页。** μP 到底是什么？讲义指向一篇易读的“μP for babies”论文。以每层宽度 $n_l$ 为变量，它基于两项断言：A1，初始化 activation 应保持 $\Theta(1)$；A2，一步梯度后 activation change 应保持 $\Theta(1)$。若单个 activation 是 $\Theta(1)$，整个向量范数应为 $\Theta(\sqrt{n_l})$。

**第 47 页。** 推导条件 A1。对深线性网络 $h_l=W_lh_{l-1}$，若 $W_l\sim\mathcal N(0,\sigma^2I)$，矩阵集中性给出谱范数趋近 $\sigma(\sqrt{n_{l-1}}+\sqrt{n_l})$，并有 $\lVert h_l\rVert_2\approx\lVert W_l\rVert_*\lVert h_{l-1}\rVert_2$。选取讲义给出的 $\sigma=\Theta(n_{l-1}^{-1/2}\min(1,\sqrt{n_l/n_{l-1}}))$，在 $\lVert h_{l-1}\rVert_2=\Theta(\sqrt{n_{l-1}})$ 的归纳假设下，得到 $\lVert h_l\rVert_2=\sqrt{n_l}+o(\sqrt{n_l})$。这是偏 worst-case 的推导，“≈”实际是上界。

**第 48 页。** 推导条件 A2。设权重更新为 $\Delta W_l$；SGD 在线性层产生 rank-one 的 loss-gradient 与 activation 外积：$\Delta W_l=-\eta_l\nabla_{h_l}\ell\,h_{l-1}^{\top}$。activation 更新展开为 $\Delta h_l=W_l\Delta h_{l-1}+\Delta W_l(h_{l-1}+\Delta h_{l-1})$。若领先阶项不抵消，归纳项为 $\Theta(\sqrt{n_l})$，要让新权重项同阶，需要 $\lVert\Delta W_l\rVert_*=\Theta(\sqrt{n_l/n_{l-1}})$；交叉项不得超过这一尺度。

**第 49 页。** A2 第二部分问怎样选择 LR 使 $\lVert\Delta W_l\rVert_*\sqrt{n_{l-1}}=\Theta(\sqrt{n_l})$。再假设 loss update 为 $O(1)$，把 $\Delta\ell$ 写成更新与梯度的内积/范数乘积，并代入 SGD 更新，可得到梯度谱尺度；最终有 $\eta_l=\Theta(n_l/n_{l-1})$。Adam 的坐标归一化改变了更新尺度，讲义旁注给出其对应关系。

**第 50 页。** μP 小结：通过 $W$ 与 $\Delta W$ 控制 activations 及其变化。初始化标准差按 $\Theta(n_{l-1}^{-1/2}\min(1,\sqrt{n_l/n_{l-1}}))$；SGD learning rate 按 $n_l/n_{l-1}$，Adam 的对应因子为 $1/n_{l-1}$。标准参数化通常使用 $1/\sqrt{n_{l-1}}$ 初始化和 $\Theta(1)$ LR。差异包括 Adam LR 随宽度变化，以及 fanout $n_l$ 小于 fanin 时的初始化差别。

**第 51 页。** 更深入看 μP：它是让超参数随 width 缩放的程序。表格分别列 embedding、attention 参数、MLP 输入/输出矩阵与 softmax linear 的初始化 variance、Adam LR 和精确 scaling；不同参数族不能套同一规则，attention scale 也使用与普通 $1/\sqrt D$ 不同的约定。

**第 52 页。** 复现 μP 的第一个问题：它是否如声称那样工作？width 增大时最佳 LR 是否保持不变？表格对 128、512、2048 宽度做 base-LR sweep；baseline μP 与 projection bias 的最佳列基本对齐，标记为能够 transfer。

**第 53 页。** 现代 LM 有许多偏离 μP 理论的组件：SwiGLU 与 squared ReLU、大小 batch、zero-attention 等初始化变化、RMSNorm gains、Lion 等特殊 optimizer、regularizer。问题是其中哪些会破坏 μP。

**第 54 页。** 可学习 RMSNorm gain 在所用 architecture 中会破坏 μP。表格显示 vector gain 让不同宽度的最佳 base LR 不再对齐；把 gain 改成 scalar 也未恢复迁移。不过移除这些 gain 几乎不损失性能。

**第 55 页。** 其他只依赖 gradient sign 的特殊 optimizer 是否能迁移？页中并列 AdamW 与 Lion 算法，并用 128、512、2048 宽度的 loss 表说明 Lion 的最佳 LR 列发生移动，因此该设置下没有 transfer。

**第 56 页。** 强 weight decay 是否破坏 μP？0.1 的 decoupled weight decay 可能是最显著的 μP 失败之一。表格中不同宽度的最佳 LR 不对齐，并标为未迁移；下方对比 L2 regularization 与 decoupled weight decay 的算法更新。

**第 57 页。** μP 是否有用？至少在一定程度上有用。整体看 μP 通常有效，而 SP 明显更不稳定；小规模 LR sweep 与 2M、40M、600M、10B 的大尺度表都支持 μP 参数化/初始化可能更容易调优，但没有把迁移说成无条件保证。

**第 58 页。** 实践 scaling 的挑战是：设置 width 等模型架构超参、设置 LR/batch 等 optimizer 超参，以及承担大规模 Chinchilla sweep 的 compute。可能的解决方案是先假设稳定性或使用 μP；在小尺度搜索最佳 LR/batch，然后保持不变或拟合其 scaling；再使用 WSD 一类 schedule 降低多预算训练成本。

## 三讲合起来的决策框架

训练端选择 N/D/recipe，推理端给 latency/memory 约束，二者共同决定“最佳模型”。compute-optimal 预训练模型未必 inference-optimal：更小且多训数据的模型可能训练更贵，却长期服务更便宜。A3 让你练训练预算决策；Lecture 10 提醒最终部署是另一个目标函数。
